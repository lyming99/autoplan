# 开发计划 · 反馈 #17

- 反馈编号：#17
- 生成日期：2026-07-02
- 反馈来源：概况界面（WorkspaceOverviewView）UI 缺陷
- 反馈原文：概况界面，执行日志右边，“循环运行中”，这个文字会溢出卡片，在卡片尺寸宽度不够时，应该用“…”省略

## 需求与背景

### 现象
在「概况」界面（`WorkspaceOverviewView`）的「执行日志」卡片头部，标题行（`.log-title-line`）右侧的状态徽标 `.log-phase-chip` 显示文本为
`循环运行中 · {CLI 后端} · {当前阶段}`（停止时为 `循环已停止 · …`）。当卡片宽度不足时，该徽标文本不会换行也不会截断，导致文字溢出卡片右边界，破坏布局。

### 根因分析（基于实际代码）
卡片头标题行 `src/renderer/styles/workspace.css:153` 定义：
```
.log-title-line { display: flex; align-items: center; gap: 12px; min-width: 0; }
```
其三个子元素的 flex 表现：
- `.log-title-line h2`（`flex: 0 0 auto`，`workspace.css:154`）——固定不收缩。
- `.log-title-line .hint`（`flex: 1 1 auto; min-width: 0; overflow: hidden; text-overflow: ellipsis; white-space: nowrap;`，`workspace.css:155`）——可收缩并已带省略号，会优先吸收多余空间。
- `.log-phase-chip`（`flex: 0 0 auto; white-space: nowrap;`，`workspace.css:156`）——**固定不收缩且不换行**，是溢出元凶。

当卡片变窄时，`.hint` 先收缩；一旦收缩到 min-content，`h2` 与 `.log-phase-chip` 仍是 `flex: 0 0 auto`，二者都不再收缩，处于最右侧的 `.log-phase-chip` 因 `white-space: nowrap` 被推出卡片右边界 → 溢出。

徽标内容来自 `WorkspaceOverviewView.tsx:98-100`，文本为动态拼接（状态词 + CLI 后端 + 阶段名），长度不定。

### 解决方案
让 `.log-phase-chip` 允许收缩并以省略号截断：
- `flex: 0 0 auto` → `flex: 0 1 auto`（允许收缩）；
- 增加 `min-width: 0; overflow: hidden; text-overflow: ellipsis;`，保留 `white-space: nowrap`。

由于 CSS `text-overflow: ellipsis` 自右向左截断，优先级最高的状态词“循环运行中”将始终可见，而靠后的 CLI 后端/阶段名等次要信息优先被“…”省略——完全契合反馈“宽度不够时用 … 省略”的诉求。纯 CSS 改动，无需改动组件 TSX，徽标圆角药丸样式（`border-radius: 999px`）保持不变。

### 影响范围
- 仅 `src/renderer/styles/workspace.css` 中 `.log-phase-chip` 规则（`workspace.css:156`）。
- 该 class 仅被 `WorkspaceOverviewView.tsx` 使用（“概况”界面执行日志卡片头部），不影响其它界面/卡片。
- 不涉及类型、构建产物结构、业务逻辑变更。
