# 修复 OpenCode 结构化计划生成失败并硬化任务拆解格式

## 需求概要

反馈 #12 指出 OpenCode CLI 在为需求 #39 生成结构化计划时失败，并要求 AutoPlan 渲染出的最终 Markdown 必须严格包含 `## 任务拆解` 章节、连续 P0NN 任务行、scope 注释以及最后的完整验收任务。当前影响范围集中在 OpenCode 结构化 PlanSpec 生成提示、PlanSpec 解析与渲染后的 Markdown 格式校验、失败诊断事件和对应回归测试，目标是让 OpenCode 只写合法 PlanSpec JSON，并由 AutoPlan 稳定渲染为符合任务解析器约束的计划文件。

## 任务拆解

- [x] P001: 定位 OpenCode 结构化 PlanSpec 失败链路 <!-- scope: src/loop/planGeneration.js,src/loop/opencodeAgent.js,src/agentCli.js -->
  - 验收要点：覆盖需求 #39 使用 OpenCode 生成结构化计划时，PlanSpec JSON 未落盘、JSON 不合法或渲染后 Markdown 不合规的主要失败分支。
  - 验收要点：确认失败事件能区分 CLI 退出失败、缺少 PlanSpec JSON 文件、PlanSpec schema 不合规和渲染后任务拆解格式不合规。
  - 验收要点：确认修复范围不改变 Claude、Codex、Oh My Pi 与内置 LLM 的既有计划生成策略选择。
- [x] P002: 强化 OpenCode 结构化计划生成提示与专用 agent 约束 <!-- scope: src/loop/planGeneration.js,src/loop/opencodeAgent.js -->
  - 验收要点：OpenCode 结构化计划生成提示明确要求只写 PlanSpec JSON 输出文件，不写最终 Markdown plan 文件，不修改业务代码。
  - 验收要点：提示明确说明最终 Markdown 由 AutoPlan 渲染，并必须满足 `## 任务拆解`、独占任务行、连续 P0NN 编号、scope 注释和最后完整验收任务的格式约束。
  - 验收要点：需求正文不明确时，OpenCode 仍应基于正文和少量必要代码上下文自行推断任务与 scope，不反问、不输出需要补充信息。
- [x] P003: 硬化 PlanSpec 渲染后的任务拆解格式校验 <!-- scope: src/loop/planRenderer.js,src/loop/structuredPlanSpec.js,src/loop/planGeneration.js,src/loop/planTaskSync.js -->
  - 验收要点：PlanSpec 渲染出的 Markdown 必须包含精确的二级标题 `## 任务拆解`，所有开发任务只能作为该章节下的顶层 checkbox 行。
  - 验收要点：每个任务行必须独占一行并严格渲染为 `- [ ] P0NN: 任务标题 `，scope 缺失时补为 `unknown`。
  - 验收要点：最终完整验收任务由 PlanSpec 规范化链路保证追加或移动到最后，并使用连续编号与 `scope: validation`。
  - 验收要点：验收要点只能作为缩进普通 bullet 渲染，不能被渲染成 checkbox、表格、引用块或代码块中的任务。
- [x] P004: 改善结构化计划失败诊断与兜底恢复 <!-- scope: src/loop/planGeneration.js,src/loop/structuredPlanSpec.js,src/loop/workspaceFiles.js -->
  - 验收要点：当 OpenCode 未写入指定 PlanSpec JSON 文件但 stdout 含合法 JSON 对象时，可恢复落盘并继续渲染。
  - 验收要点：当 stdout 只包含 Markdown、寒暄文本或非 JSON 内容时，失败事件记录清晰原因、日志路径、PlanSpec 目标路径和 provider 信息。
  - 验收要点：当 PlanSpec JSON 合法但渲染后的 Markdown 不满足任务拆解格式时，按计划生成失败处理，不插入 plans，不同步 plan_tasks。
  - 验收要点：失败状态写回需求或反馈记录，便于界面展示 OpenCode 结构化计划失败原因和日志位置。
- [x] P005: 补充 OpenCode 结构化计划回归覆盖 <!-- scope: src/loop/planGeneration.test.js,src/loop/opencodeAgent.test.js,src/loop/structuredPlanSpec.test.js,src/loop/planTaskSync.js,scripts/smoke-test.js -->
  - 验收要点：覆盖 OpenCode 结构化模式成功生成 PlanSpec JSON 后，最终 Markdown 含 `## 任务拆解` 和连续 `- [ ] P001: ... ` 任务行。
  - 验收要点：覆盖缺少 scope 的任务被规范化为 `unknown`，最后完整验收任务被规范化为 `scope: validation` 且编号连续。
  - 验收要点：覆盖 OpenCode 只输出 stdout JSON 的恢复路径，以及输出 Markdown 或非法 JSON 时的失败路径。
  - 验收要点：覆盖渲染后的任务拆解可被同步到 plan_tasks，且验收要点不会被误解析为开发任务。
- [x] P006: 更新结构化计划格式说明 <!-- scope: README.md,docs/architecture.md -->
  - 验收要点：文档说明外部 CLI 结构化计划生成阶段产物是 PlanSpec JSON，最终 Markdown 由 AutoPlan 渲染。
  - 验收要点：文档列出最终 Markdown 的硬性任务拆解格式，包括 `## 任务拆解`、P0NN 连续编号、scope 注释、unknown scope 和 validation scope。
  - 验收要点：文档明确运行测试、构建和最终验收不应拆成普通开发任务，最终验收由 PlanSpec 的 finalValidation 与最后的完整验收任务承载。
- [x] P007: 完整验收 <!-- scope: validation -->
  - 验收要点：最终验收命令为 npm run check && npm test && npm run smoke
  - 验收要点：OpenCode 结构化计划生成路径能够为需求或反馈生成合法 PlanSpec JSON，并渲染为符合 AutoPlan 任务拆解硬性格式的 Markdown。
  - 验收要点：最终 Markdown 必须包含精确的 `## 任务拆解` 章节，任务行必须连续编号并使用 `- [ ] P0NN: 任务标题 ` 格式，最后一项为 `scope: validation` 的完整验收任务。
  - 验收要点：PlanSpec 缺字段、非法 JSON、未落盘、stdout 仅 Markdown、渲染后任务拆解不合规等异常均被明确拦截并记录失败原因，不产生空任务或畸形 plan_tasks。
  - 验收要点：既有 Claude、Codex、Oh My Pi、内置 LLM 结构化计划生成、任务同步、计划阅读和常规 smoke 流程无回归。
