# 执行中计划支持修改 Codex 思考深度

## 需求概要

当前计划执行配置会在计划生成时写入 plans 快照，任务执行和验收会优先读取该快照；因此项目设置里的计划执行 Codex 思考深度变更无法稳定影响已经生成、正在执行的计划。本次需求需要补齐运行中计划的思考深度更新链路，让用户在计划执行期间调整 Codex 思考深度后，后续任务、重试和最终验收使用新的深度，同时保留事件记录和前端状态同步。
