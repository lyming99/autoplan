# 反馈 #21 开发计划：聊天窗口对话气泡右对齐

## 背景

反馈 #21 要求“聊天窗口，对话气泡要右对齐”。结合当前项目源码，实际聊天窗口由 React 组件 `src/renderer/components/workspace/ChatView.tsx` 渲染，布局样式集中在 `src/renderer/styles/workspace.css` 的 Chat 对话模块。旧版静态设计原型 `ui/chat.html` 仍保留一套聊天气泡样式与说明，可能被用于截图、设计对照或人工验收参考。

当前运行界面的用户消息 DOM 结构为 `.chat-message.chat-message--user > .chat-message__content > .chat-message__bubble`。样式里 `.chat-message` 是 `width: 100%` 的 flex 行，`.chat-message--user` 同时设置了 `justify-content: flex-end` 与 `flex-direction: row-reverse`。在 `row-reverse` 主轴方向下，`flex-end` 的视觉落点容易与预期相反，导致用户对话气泡没有稳定贴近消息列右侧。该反馈应优先解释为：用户发送的对话气泡需要右对齐，AI 回复与工具卡片继续左对齐，系统提示继续居中。

## 设计约束

- 只调整聊天消息布局，不改变聊天发送、流式输出、模型配置、对话列表、历史消息和后端协议。
- 用户消息气泡右对齐时，气泡内部文本仍保持自然左对齐，长文本按现有规则换行，不溢出消息列。
- AI 回复、工具调用卡片、系统提示的既有对齐语义保持不变：AI/工具左对齐，系统提示居中。
- 运行界面样式以 `src/renderer/styles/workspace.css` 为准；`ui/chat.html` 作为静态原型同步更新，避免原型与实际界面表达冲突。
- 视觉变更需要覆盖宽屏、窄屏和移动断点，避免在 `max-width: 960px`、`760px`、`420px` 断点下退回左对齐或撑满异常。
