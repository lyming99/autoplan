# 需求 #12 开发计划：增加 Claude CLI 支持

## 背景

当前 AutoPlan 的计划生成、任务执行、修复与验收流程主要通过 `codex exec` 调用 Codex CLI。需求 #12 要求“增加 claude cli 支持”，本计划默认目标是：在不破坏现有 Codex 默认行为、任务上下文复用、并发 scope 控制和日志展示的前提下，为项目级配置增加 Claude CLI 作为可选执行后端。

## 设计约束

- 默认后端保持 Codex，历史项目和旧数据库升级后无需重新配置即可继续运行。
- Claude CLI 作为项目级选项保存，允许不同项目选择不同 CLI 后端。
- Codex 的 session 复用能力仅在 Codex 后端启用；切换到 Claude 时不得把历史 `codex_session_id` 传给 Claude CLI。
- Claude CLI 调用必须使用当前项目 `workspace_path` 作为工作目录，并接收与 Codex 相同语义的 prompt。
- CLI 不存在、认证失效、非零退出、用户停止等状态必须回写到现有事件、活动日志和任务状态链路。

## 任务拆解

- [x] P001: 增加项目级 CLI 后端配置 <!-- scope: src/database.js,src/renderer/types.ts -->
  - 验收要点：`project_states` 或等价配置表新增 CLI 后端字段，至少支持 `codex` 与 `claude`，历史数据默认值为 `codex`。
  - 验收要点：配置字段通过现有迁移机制兼容旧数据库，应用启动、项目快照和项目列表不会因缺少字段报错。
  - 验收要点：前端类型新增 CLI 后端、命令路径或命令名等字段，`Project`、`Snapshot`、`ActiveOperation` 等相关类型不出现类型缺口。
  - 验收要点：非法后端值会被规范化或拒绝保存，不会进入执行链路导致 spawn 未知命令。

- [ ] P002: 打通配置保存与 IPC 链路 <!-- scope: src/main.js,src/preload.js,src/loopService.js -->
  - 验收要点：项目创建、项目更新和运行配置更新接口能够读写 CLI 后端配置，并随 `snapshot` 返回最新值。
  - 验收要点：`loop.configure` 保留现有 `workspacePath`、`intervalSeconds`、`validationCommand` 行为，同时支持更新 CLI 后端配置。
  - 验收要点：配置变更后不需要重启应用即可影响后续计划生成、任务执行、修复和验收调用。
  - 验收要点：删除项目、停止循环、手动运行任务等既有 IPC 行为不因新增配置字段发生回归。

- [x] P003: 抽象通用 Agent CLI 执行器 <!-- scope: src/loopService.js,src/agentCli.js,src/codexActivity.js -->
  - 验收要点：将当前写死的 `runCodex`/`codex exec` 参数组装抽象为按后端选择的执行器，保留兼容包装以减少现有调用点改动风险。
  - 验收要点：Codex 后端生成的命令参数、工作目录、stdin prompt、日志文件、last message 文件和退出码处理与改造前保持一致。
  - 验收要点：执行器统一返回 `exitCode`、`logFile`、`lastFile`、后端名称、会话信息、活动摘要和可读错误信息。
  - 验收要点：停止任务时仍能终止当前 CLI 子进程及其子进程树，且多任务并发执行时各自登记独立进程。

- [x] P004: 实现 Claude CLI 非交互调用 <!-- scope: src/agentCli.js,src/loopService.js -->
  - 验收要点：Claude 后端默认调用可配置命令名，默认值为 `claude`，并使用 Claude CLI 的非交互/打印模式执行完整 prompt。
  - 验收要点：Claude 调用在项目工作区内运行，prompt 传递方式不经过危险字符串拼接，避免路径或需求内容触发命令注入。
  - 验收要点：Claude stdout/stderr 会写入现有日志文件，最终回复会写入 last message 文件，非零退出会让对应计划或任务进入现有失败/重试流程。
  - 验收要点：Claude 后端不使用 `codex_session_id`；如果未来 Claude CLI 返回稳定会话标识，也必须与 Codex 会话字段隔离。

- [x] P005: 适配多 CLI 活动日志与展示文案 <!-- scope: src/renderer/components/CodexLog.tsx,src/renderer/pages/WorkspacePage.tsx,src/renderer/types.ts,src/loopService.js -->
  - 验收要点：活动日志组件能根据当前后端显示 `Codex` 或 `Claude`，等待文案、状态摘要和错误信息不再固定写死为 Codex。
  - 验收要点：Codex 的结构化 activity 解析继续可用；Claude 输出无法解析为结构化事件时仍显示原始日志尾部和基础状态。
  - 验收要点：运行中概览、事件流、任务卡片和 active operation 元信息展示当前 CLI 后端，便于用户判断正在使用哪个工具。
  - 验收要点：已有 Codex 会话标签、恢复会话、回退新建会话等文案在 Codex 后端不回归；Claude 后端不展示误导性的 Codex 会话恢复状态。

- [ ] P006: 增加项目设置 UI 入口 <!-- scope: src/renderer/pages/ProjectsPage.tsx,src/styles.css,src/renderer/pages/WorkspacePage.tsx -->
  - 验收要点：项目编辑或创建界面提供 CLI 后端选择，至少包含 `Codex CLI` 和 `Claude CLI` 两个选项。
  - 验收要点：界面清楚提示 Claude CLI 需要用户本机已安装并完成认证，保存后能在项目列表或工作区概览看到当前后端。
  - 验收要点：切换后端时只影响后续执行，不主动修改历史 plan 文件、任务状态或已有 Codex session id。
  - 验收要点：样式适配现有项目配置表单，在窄屏和当前 Electron 窗口尺寸下不遮挡主要操作按钮。

- [x] P007: 补充 smoke 覆盖多后端执行 <!-- scope: scripts/smoke-test.js -->
  - 验收要点：新增 smoke 用例覆盖默认配置为 Codex，确保历史项目升级后 `agentCliProvider` 或等价字段为 `codex`。
  - 验收要点：通过 stub 执行器验证选择 Claude 后，计划生成和任务执行会走 Claude 后端，不会传入 `codexSessionId`。
  - 验收要点：通过 stub 执行器验证选择 Codex 后，现有 session 复用、resume 失败回退、新建会话等断言继续通过。
  - 验收要点：验证 CLI 后端配置通过 IPC 保存、快照返回和 UI 类型约束保持一致。

- [ ] P008: 完成检查与使用说明更新 <!-- scope: README.md,ui/README.md,package.json -->
  - 验收要点：README 或 UI 说明补充 Claude CLI 支持范围、安装/认证前置条件、项目级切换方式和默认仍为 Codex 的兼容说明。
  - 验收要点：若新增测试脚本或检查命令，需要同步更新 `package.json`；若复用现有命令，则不得无意义修改脚本。
  - 验收要点：`npm run check` 通过，覆盖 TypeScript、主进程 JS、preload 和 smoke 相关静态检查。
  - 验收要点：`npm run smoke` 通过，确认 Codex 旧链路和 Claude 新链路均满足核心流程断言。

## 总体验收标准

- 兼容性：未配置过 CLI 后端的历史项目继续默认使用 Codex，计划生成、任务执行、停止、重试、验收和并发 scope 控制保持原行为。
- 可配置性：用户可以在项目级配置中选择 Codex CLI 或 Claude CLI，保存后无需重启即可用于后续自动循环和手动任务执行。
- Claude 可用性：选择 Claude 后，AutoPlan 能在项目工作区调用 Claude CLI 非交互执行 prompt，并正确采集 stdout/stderr、last message、退出码和错误信息。
- 会话隔离：Codex session 复用能力不回归；Claude 后端不会错误复用或污染 `codex_session_id`，不同任务、不同项目、不同后端之间上下文不串扰。
- 可观测性：运行中日志、事件流、任务卡片和概览能明确显示当前 CLI 后端，CLI 缺失、认证失败、命令失败等问题有可读提示和日志可追踪。
- 验证通过：完成后 `npm run check` 与 `npm run smoke` 均通过；如本机未安装 Claude CLI，smoke 必须通过 stub 或 mock 覆盖核心分支，真实 Claude 调用可作为手动验收项。
