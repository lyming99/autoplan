# 支持自定义新建项目默认 CLI

## 需求概要

反馈 #44 希望新建项目时不要固定默认使用 Codex CLI，而是允许用户自定义新建项目的默认 CLI。当前项目创建弹窗的 emptyDraft、工作区控制器初始状态和表单归一化都以 codex 作为硬编码默认值，后端 project_states 也保持 codex 兼容默认。计划是在不破坏既有 Codex 回退的前提下，为项目创建入口增加可持久化的新项目默认 CLI 偏好，并让新建项目、项目设置和需求/反馈计划生成默认值继续按项目状态一致生效。
