# 验收模块支持重做并附加说明

## 需求概要

当前验收模块已支持对已完成计划和任务进行人工验收、取消验收以及批量操作，但当人工验收不通过时，用户缺少从验收页直接发起返工的入口，也无法把不通过原因或补充要求传递给后续执行。需要在验收模块增加“重做”能力：对待验收或已验收的计划/任务可触发返工，允许输入可选补充内容，后端清理人工验收态并把目标重新放回可执行队列，同时记录事件并保持现有验收/取消验收行为不回归。
