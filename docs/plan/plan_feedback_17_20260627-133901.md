# 反馈 #17 开发计划：事件提示类型中文化

## 背景

反馈 #17 指出：“这类提示换成中文：scan.done，feedback.created，plan.generated”。当前事件流中部分系统事件会以英文事件类型作为标题展示，例如扫描完成、反馈创建、计划生成等状态提示。虽然事件正文大多已经是中文，但标题仍直接暴露内部事件 key，影响中文界面的一致性与可读性。

## 设计约束

- 本计划只处理事件提示的中文展示，不改变事件类型入库值、事件写入流程、扫描逻辑、反馈创建逻辑和计划生成逻辑。
- `scan.done`、`feedback.created`、`plan.generated` 必须保留为内部事件类型，避免破坏搜索、筛选、统计、历史数据兼容和 smoke 断言。
- 中文化应优先在统一事件展示格式化层完成，确保事件列表、搜索文本和相关 UI 复用同一套展示结果。
- 已有任务事件的专用展示、徽标、语气样式和元信息格式不应被本次普通系统事件中文化影响。
- 不要把运行测试、回归、构建拆成普通开发任务；最终检查统一放入最后的“完整验收”节点。

## 总体验收标准

- 中文展示：事件列表中 `scan.done`、`feedback.created`、`plan.generated` 的用户可见标题分别中文化为“扫描完成”、“反馈已创建”、“计划已生成”。
- 内部兼容：数据库事件 `type` 仍保留原始英文 key，已有历史事件、搜索命中、统计逻辑和 smoke 断言不因中文化而失效。
- 搜索一致：事件搜索结果优先展示中文标题，同时保留对原始事件 key 的搜索命中能力。
- 兜底稳定：未知事件类型继续显示原始 key，不出现空标题、异常报错或误用中文标题的情况。
- 影响可控：任务事件、计划列表、反馈创建、扫描循环和计划生成流程不因本次展示文案调整发生行为变化。
- 最终验收命令：执行 `npm run check && npm run smoke`；验收范围为事件展示格式化、事件搜索结果、反馈创建事件、扫描完成事件、计划生成事件和现有 smoke 主流程；通过标准为命令退出码为 0，且手动验证三类事件提示均不再以英文 key 作为主要标题展示。
