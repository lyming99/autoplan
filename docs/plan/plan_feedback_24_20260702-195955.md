# 反馈 #24 开发计划：聊天窗口消息与工具调用简化

## 背景

反馈要求聚焦对话模块的聊天窗口体验：AI 回复内容不再像卡片气泡一样被边框包住，工具调用也要尽可能简约，具体调用参数和结果改为点击后在 dialog 中查看；聊天内容区域应尽量占满可用宽度，视觉上只区分 AI 左对齐、用户右对齐；单独用户消息仍需要保留明确的气泡效果。

结合当前代码，聊天渲染集中在 `src/renderer/components/workspace/ChatView.tsx`，样式集中在 `src/renderer/styles/workspace.css`。现状是 AI 回复使用 `.chat-message__bubble--markdown` 灰底边框气泡，工具调用使用 `.chat-tool-card` 边框卡片并在消息流内展开详情，和本次“简约、无边框、详情进 dialog”的目标不一致。已有共享 `Modal` 组件可复用，不需要改动后端流式事件、消息存储或工具调用协议。

## 设计约束

- 保持 `ChatMessage.role === 'tool'` 的现有消息模型和流式 `tool_start/tool_result` 拼装逻辑不变，只调整渲染和交互。
- AI 回复、工具调用摘要、思考区都属于 AI 侧内容，默认左对齐并使用整行可用宽度，不再套大面积气泡边框。
- 用户消息继续右对齐，保留气泡背景、圆角和适当宽度限制，长文本自然换行且不强制右对齐正文。
- 工具调用摘要只展示名称、状态、必要的极短参数摘要或结果状态；参数、结果、空结果、加载中等完整信息进入 dialog。
- 移动端、深色主题、Markdown 代码块和表格不能出现横向撑破、文本遮挡或按钮内容溢出。

## 总体验收标准

- 最终验收命令：`npm test && npm run check && npm run build`。
- 验收范围：对话模块聊天窗口的消息渲染、工具调用摘要与详情 dialog、AI/用户/工具/系统消息对齐、响应式样式、深色主题，以及聊天 UI 源码级回归断言。
- 通过标准：自动化命令全部通过；AI 回复和工具调用不再呈现边框卡片；工具详情只通过 dialog 查看；AI 内容在聊天列内占满宽度并左对齐；用户消息右对齐并保留气泡效果；窄屏和深色主题下无明显溢出、遮挡或不可读状态。
