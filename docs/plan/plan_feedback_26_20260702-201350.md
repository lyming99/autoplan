# 反馈 #26 开发计划：工作区搜索弹层改为全局弹层

> 反馈原文：搜索框 popup，应该全局 popup，目前被左侧导航遮挡住了。

## 背景 / 根因分析

工作区布局为 `.workspace-shell` 的 CSS Grid：左列 `.sidebar`（248px）+ 右列 `.workspace-main`（`1fr`，且设置了 `overflow: hidden`）。
搜索结果弹层 `SearchResults`（类名 `.search-results-popup`）当前以 `position: absolute` 渲染在锚点 `.workspace-search-popover-anchor`（`position: relative; z-index: 30`）内部，而锚点位于 `.workspace-main` 之内。

由此产生两个问题，导致弹层“被左侧导航遮挡”：

1. **被裁剪**：`.workspace-main { overflow: hidden }` 会裁剪溢出主区左边界（即侧边栏右边界）的内容。弹层宽度可达 760px（`width: min(760px, calc(100vw - 48px))`）且以搜索框右边缘对齐向左展开，在较多分辨率下其左侧会越出主区左边界，被精确地在侧边栏边界处裁掉，表现为“被左侧导航挡住”。
2. **层级受限**：弹层处于锚点的层叠上下文（z-index 30）内，作为 `.workspace-main` 子树参与绘制，无法稳定覆盖独立的左侧导航列。

工程上最稳妥、且与用户“全局 popup”诉求一致的修复方式是：用 React Portal（`createPortal`）把搜索结果弹层渲染到 `document.body`，并以 `position: fixed` 配合搜索框锚点的 `getBoundingClientRect()` 定位，彻底脱离 `.workspace-main` 的 `overflow:hidden` 与侧边栏的网格/层叠区域。这与项目中既有的 `.modal-mask`（`position: fixed; z-index: 100`）思路一致。

关键风险点：弹层 Portal 化后，DOM 不再是锚点 `.workspace-search-popover-anchor` 的后代，现有“点击外部关闭”逻辑 `searchPopupRef.current.contains(target)`（`WorkspacePage.tsx` 的 `handlePointerDown`）将恒为 false——点击弹层内部会被误判为外部点击而立即关闭。本计划同步修复该判定。

## 任务拆解

- [x] P001: 搜索结果弹层改为通过 Portal 渲染到 document.body 的全局弹层 <!-- scope: src/renderer/components/SearchResults.tsx -->
  验收要点：
  - 使用 `react-dom` 的 `createPortal` 将 `.search-results-popup` 渲染到 `document.body`，使其不再是 `.workspace-search-popover-anchor` 的 DOM 后代，从而脱离 `.workspace-main { overflow: hidden }` 的裁剪与左侧导航的网格/层叠区域。
  - 弹层改为 `position: fixed`，其 `top` / 右对齐基准 / `width` 由传入的锚点坐标（基于 `.workspace-search-popover-anchor` 的 `getBoundingClientRect()`）计算，视觉上仍紧贴搜索框正下方、与搜索框右对齐。
  - `open` 为 false 或 `searchState.query.isEmpty` 时继续返回 `null`，不向 DOM 挂载任何节点。
  - 保留现有 `role="dialog"`、`aria-label="统一搜索结果"`、`aria-live="polite"` 与内部 Esc 关闭（`handleKeyDown`）等无障碍与交互行为不变。
  - 窄屏（≤760px）下改为左对齐、宽度受限于视口，`position: fixed` 下不产生水平溢出或错位。

- [x] P002: 在 WorkspacePage 计算并传入锚点坐标，并修复 Portal 后的外部点击关闭判定 <!-- scope: src/renderer/pages/WorkspacePage.tsx -->
  验收要点：
  - 新增弹层定位坐标状态：在弹层打开、`window` 的 `scroll`/`resize` 时，基于锚点 ref（现有 `searchPopupRef`，仍挂在 `.workspace-search-popover-anchor`）读取 `getBoundingClientRect()` 并更新坐标，随 props 传入 P001 的 `SearchResults`。
  - 修复 `handlePointerDown` 外部点击关闭判定：Portal 后弹层不在锚点子树内，必须同时排除弹层自身节点（例如为弹层新增 ref 或通过其稳定 class / `data-*` 属性判定），确保“点击弹层内部”不被判定为外部点击。
  - 点击弹层内的结果项 / 分组 / 清空按钮：不因外部点击逻辑提前关闭（结果项的关闭仍由其自身 `onClick → onClose` 显式触发）。
  - 点击搜索框/锚点区域不关闭；点击左侧导航、`.workspace-main` 空白区域关闭。
  - 滚动主内容区或调整窗口大小时，弹层位置随锚点同步刷新，不错位、不残留。
  - Esc 关闭与既有 `useEffect` 中的监听卸载逻辑保持稳定，无新增监听泄漏。

- [x] P003: 调整全局弹层 CSS 定位与层级 <!-- scope: src/styles.css -->
  验收要点：
  - `.search-results-popup` 由 `position: absolute; right: 0` 调整为适配 Portal 的 `position: fixed`；`top/right/width` 等定位基值改为兜底（实际坐标由内联 style 覆盖），避免与 P001 内联定位冲突。
  - 弹层 `z-index` 提升至高于侧边栏与 topbar 锚点（如 90 量级），并低于 `.modal-mask`（100），保证打开 PlanReaderModal 等模态时模态仍位于搜索弹层之上。
  - 保留圆角、阴影、`.search-results-body` / `.search-result-groups` 内部布局与 `[data-theme="dark"]` 暗色阴影规则不回归。
  - 复核 `@media (max-width: 900px)`、`@media (max-width: 760px)` 响应式分支在 fixed 定位下仍合理占位，无水平裁剪。

- [x] P004: 补充测试代码覆盖全局弹层 Portal 化与外部点击判定 <!-- scope: src/renderer/pages/WorkspacePage.test.tsx,src/renderer/components/SearchResults.test.tsx -->

  - 查询清空 / `open=false` 时，`document.body` 下不残留弹层节点。
  - 弹层打开时点击其内部结果项，不会因外部点击逻辑被提前关闭（仅由结果项自身回调关闭）。
  - 弹层打开时点击左侧导航或主区空白处会关闭；Esc 会关闭。
  - 窗口 `resize` 后弹层定位坐标被刷新（锚点变化时坐标随之更新）。
