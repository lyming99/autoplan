# 需求 #36 开发计划：根据设计稿优化设置界面与任务界面

## 需求概述

根据 `ui` 目录中的设计稿，对 AutoPlan 的设置界面与任务界面做视觉和交互优化，使实际 React 界面在布局层级、信息密度、卡片样式、状态表达、按钮与表单控件呈现上与设计稿保持一致。重点参考 `ui/settings.html` 与 `ui/tasks.html`，并确保优化不破坏现有工作区导航、搜索定位、任务筛选、计划阅读、任务状态展示与设置持久化行为。

## 设计约束

- 以 `ui/settings.html`、`ui/tasks.html` 作为视觉和交互参考，允许在实现中复用现有 React 组件结构，但最终观感需对齐设计稿。
- 设置界面优化应保留现有 Scope 文件打开方式配置、命令输入、帮助说明和保存反馈逻辑，不改变本地存储键与数据结构。
- 任务界面优化应保留现有任务筛选、按计划分组、状态统计、定位高亮、计划关联和空状态逻辑，不改变任务数据来源。
- 样式调整优先沉淀到工作区相关样式文件，避免在组件内引入大段内联样式。
- 若设计稿与当前实现存在命名或内容差异，应优先保持现有产品语义与数据字段，仅调整呈现层。
- 完整验收节点只负责最终验证，不承载普通开发实现。

## 任务拆解

- [ ] P001: 对齐设计稿视觉规范与页面结构 <!-- scope: ui/settings.html,ui/tasks.html,src/renderer/styles/workspace.css,src/renderer/styles/components.css -->
  - 验收要点：
    - 明确设置界面与任务界面的布局层级、间距、卡片、按钮、表单、状态标签和空状态样式差异。
    - 确认需要复用或新增的 CSS 类集中在工作区样式体系内，避免影响项目列表、需求、反馈、事件等无关界面。
    - 形成后续设置页与任务页可共同使用的基础视觉规则，例如卡片边框、标题区、辅助说明和操作区布局。

- [ ] P002: 优化设置界面布局与表单呈现 <!-- scope: src/renderer/components/workspace/WorkspaceSettingsView.tsx,src/renderer/styles/workspace.css -->
  - 验收要点：
    - 设置页整体布局与 `ui/settings.html` 对齐，包含清晰的标题区、配置卡片、说明卡片和合理的内容宽度。
    - Scope 文件打开方式的选择控件、命令输入框、保存按钮和保存反馈在视觉层级上清晰可辨。
    - `system`、`vscode`、`command` 三种模式切换后，命令输入区域显示逻辑与原有行为一致。
    - 保存设置后仍能正确写入现有配置状态，成功提示或反馈文案不被样式改造遮挡。

  - 验收要点：
    - 任务页顶部筛选、统计信息和列表区域与 `ui/tasks.html` 的信息层级保持一致。
    - 任务按计划分组的标题、摘要、状态计数和任务数量展示更易扫描，不丢失原有字段。
    - 任务状态筛选按钮保留原有筛选语义，切换后列表数量、空状态文案和选中态展示正确。
    - 搜索定位到任务页时，目标任务所在分组展开与高亮行为保持可用。

- [ ] P004: 优化任务卡片状态与操作信息 <!-- scope: src/renderer/components/PlanLists.tsx,src/renderer/styles/workspace.css,src/renderer/styles/components.css -->
  - 验收要点：
    - 单个任务卡片的编号、标题、状态、耗时、更新时间、scope 与日志信息按设计稿形成清晰层级。
    - 不同任务状态的标签颜色和视觉权重可区分，且与现有状态枚举兼容。
    - 长标题、长 scope、长日志路径在窄宽度下不破坏布局，必要时换行或截断仍可阅读。
    - 任务为空、计划无任务、筛选无结果等空状态与任务页整体风格一致。

- [ ] P005: 同步静态设计稿说明与实现备注 <!-- scope: ui/README.md,docs/plan/plan_requirement_36_20260627-220305.md -->

    - 开发记录明确本次优化参考的设计稿文件和主要影响模块，便于后续 UI 回归定位。

## 总体验收标准

- 视觉一致性：设置界面与任务界面的整体结构、卡片层级、间距、按钮、表单控件、状态标签和空状态与 `ui/settings.html`、`ui/tasks.html` 保持一致或具备合理实现映射。
- 功能完整性：设置保存、打开方式切换、任务筛选、任务分组、任务详情展示、搜索定位和工作区 Tab 切换均保持现有行为。
- 影响范围：样式改造不应明显影响需求、反馈、事件、概览、项目列表等非本次目标页面；共享样式变更需保持向后兼容。
- 最终验收命令：执行 `npm run check` 与 `npm run build`；两条命令均退出码为 0，且无新增类型错误、语法错误或构建错误。
- 手动验收范围：在本地应用中打开任一工作区，检查“设置”和“任务与计划”界面；切换任务筛选状态、定位任务、修改并保存设置，确认视觉与交互均符合预期。
