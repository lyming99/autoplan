# 反馈 #8 开发计划

## 背景与目标

反馈日志暴露两个独立问题：

1. `src/renderer/hooks/useTheme.ts` 中包含 JSX 返回值，但文件扩展名是 `.ts`，Vite/OXC 按普通 TypeScript 解析时在 `<ThemeContext.Provider value={value}>` 处报 `Expected > but found Identifier`。
2. MCP HTTP 服务默认监听 `127.0.0.1:43847`，端口被占用时启动失败并向事件流/控制台输出端口占用错误。当前已有错误分类与配置入口，但默认启动场景缺少自动恢复策略。

目标是恢复前端构建与开发服务器可用性，并让 MCP 默认 HTTP 端口被占用时具备可预期的降级行为，同时保留用户显式配置固定端口时的明确错误提示。

## 进度区

- 当前状态：待开发
- 已完成：已定位 `useTheme.ts` JSX 解析失败根因；已定位 MCP HTTP 监听失败路径在 `src/mcpServer.js:startHttp()` 与 `describeListenError()`
- 待执行：按任务拆解实现、补充测试、完成整体验收
