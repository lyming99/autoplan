# 需求 #38 开发计划：实现 zcode CLI 接入

## 背景

需求 #38 要求“实现zcode cli接入”。当前 AutoPlan 已支持 Codex CLI 与 Claude CLI，计划生成、任务执行、失败修复和完整验收均通过统一的 Agent CLI 配置链路传递后端、命令路径与执行元信息。zcode 接入应复用现有 CLI 后端抽象，在项目级配置、单条需求/反馈配置、计划级快照、任务事件、日志展示和 smoke 覆盖中形成完整闭环。

## 设计约束

- 默认后端继续保持 Codex，历史项目、历史需求/反馈和历史计划缺少后端字段时仍按 Codex + `medium` 思考深度降级展示。
- zcode 作为独立后端接入，不复用 Codex 的 session resume、`model_reasoning_effort` 参数或 Codex 专属活动解析；Codex 会话字段仅在 Codex 后端生效。
- zcode 默认命令建议为 `zcode`，自定义命令路径继续沿用现有 `agentCliCommand` 字段；具体非交互参数必须以实现时本机 `zcode --help` 或官方文档确认结果为准，避免调用交互模式导致任务挂起。
- 现有 Codex 与 Claude 行为不能回退，包括命令参数、stdin prompt、日志输出、错误提示、计划级快照和 UI 文案。
- 不新增不必要的数据库字段；现有 CLI 后端字段为文本类型时，优先通过归一化、展示和测试覆盖完成扩展。
- 不要把运行测试、回归、构建拆成普通开发任务；最终检查统一放入最后的“完整验收”节点。

## 总体验收标准

- 后端完整性：AutoPlan 支持 `codex`、`claude`、`zcode` 三类 CLI 后端，默认仍为 Codex，未知值仍安全降级。
- 执行正确性：zcode 使用独立的非交互执行参数和输出读取策略，不携带 Codex session、Codex resume 或 Codex 思考深度参数。
- 配置闭环：项目级设置、需求/反馈单条覆盖、计划生成快照、任务执行上下文和事件 meta 均能保存并传递 `zcode`。
- 展示一致性：设置页、输入区、概览、事件流、任务/计划列表和计划阅读器均显示 ZCode CLI，且非 Codex 后端不展示 Codex 思考深度。
- 兼容稳定：Codex 和 Claude 原有命令参数、历史数据降级、空验收命令、并发执行、失败修复和完整验收重试不发生回退。
- 文档可用：README 清楚说明 ZCode CLI 的安装/认证前置条件、默认命令、自定义命令路径、非交互模式要求和与 Codex 专属能力的差异。
- 最终验收命令：执行 `npm run check && npm run smoke && npm run build`；验收范围为 CLI 适配层、配置持久化、前端展示、任务执行/修复/验收链路、文档和现有主流程；通过标准为命令退出码均为 0，且手动验证 zcode 成功与缺失场景均符合预期。
