# 修复 Windows OpenCode CLI 多行计划提示词仅传入首行

## 需求概要

反馈 #6 所附的三次 OpenCode 计划生成日志均显示，autoplan-plan agent 实际只接收到结构化计划 prompt 的首行“你是需求整理与开发计划生成者。”，随后因缺少需求正文、PlanSpec 输出路径和格式契约而遍历仓库、等待补充信息，最终未生成计划。当前 Windows 执行路径会把多行位置参数直接拼入 `cmd.exe /d /s /c call opencode.cmd ...`；现有 spillover 只按命令行总长度触发，无法保护长度未超限但含 CR/LF 的 PlanSpec prompt。需要在 OpenCode 参数投递层识别 Windows `.cmd` shim 不安全的多行参数，改用完整内容落盘并通过单行指针与 `-f` 附件传递，同时以贴近真实结构化计划 prompt 的单元测试防止再次只传首行。
