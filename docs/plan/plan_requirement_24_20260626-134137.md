# 需求 #24 开发计划：任务列表按计划分组并按完成时间倒序展示

## 背景

当前工作区的任务列表以扁平列表展示，主要按计划创建时间和任务顺序排列。需求希望任务列表按照所属计划分组展示，分组标题使用计划标题，并按完成时间排序，最新完成的计划或任务排在列表前面，便于用户优先查看最近完成的工作。

## 设计约束

- 任务列表仍应保留现有状态筛选能力，包括全部、进行中、队列中、已完成等筛选项。
- 分组标题优先使用计划标题；当计划标题缺失时，应使用计划文件名或可读兜底文案，避免出现空标题。
- 完成时间排序优先使用任务 `finished_at`；没有完成时间的任务按运行开始时间、更新时间或原计划顺序兜底，保证未完成任务也能稳定展示。
- 计划分组的整体排序应以组内最新完成时间或最新任务时间为准，最新的分组排在最前面。
- 每个分组内的任务也应按完成时间倒序展示；同一时间或缺少时间时保持稳定的任务顺序，避免列表闪烁。
- 数据层应尽量在现有 snapshot 中补充必要字段，不引入新的持久化表结构，除非实现时发现现有字段无法支持稳定排序。
- UI 样式应与现有任务卡片、筛选 tab、计划阅读入口保持一致，不破坏当前执行、停止、预览计划等操作。

## 总体验收标准

- 任务列表不再以全局扁平列表呈现，而是按照所属计划分组，分组标题使用计划标题并具备文件路径或“未命名计划”兜底。
- 任务列表排序以完成时间为核心：最新完成的计划分组排在最前，组内最新完成的任务排在最前；缺少完成时间时使用开始时间、更新时间或原顺序稳定兜底。
- 状态筛选、搜索过滤、执行任务、停止任务、打开计划全文等现有功能在分组展示后仍可正常使用。
- 边界数据不会导致 UI 报错，包括缺少计划标题、缺少完成时间、无任务、多计划混合状态等情况。
- 最终验收命令：`npm run check`、`npm run smoke`，并在 Electron 应用中手动验证任务列表分组、倒序排序、筛选和操作按钮。
- 通过标准：所有验收命令退出码为 0；手动验证中最新完成项排在前面；任务按计划标题分组；筛选与操作行为无回归；开发者工具无新增运行时错误。
