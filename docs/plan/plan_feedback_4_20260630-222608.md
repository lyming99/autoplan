# 开发计划：增加深色主题

## 需求分析

当前 AutoPlan 仅支持浅色主题，所有设计 token（CSS 变量）硬编码在 `:root` 中，`color-scheme` 固定为 `light`。需要新增深色主题支持，允许用户在浅色/深色/跟随系统 三种模式之间切换，并将偏好持久化。

### 关键技术决策

1. **实现方式**：使用 `data-theme` 属性挂载在 `<html>` 上，通过 CSS 属性选择器覆盖设计 token。这是最简方案，无需引入 CSS-in-JS 或第三方主题库。
2. **三模式支持**：浅色（light）/ 深色（dark）/ 跟随系统（auto），默认 `auto`。
3. **持久化**：通过 `localStorage` 存储用户选择，应用启动时读取并应用。
4. **切换入口**：在设置视图（WorkspaceSettingsView）新增"外观"配置卡片；同时在侧栏底部提供快捷切换按钮。

### 影响范围

- `src/renderer/styles/layout.css` — 新增 `[data-theme="dark"]` 设计 token 覆盖
- `src/renderer/components/workspace/WorkspaceSettingsView.tsx` — 新增外观设置卡片
- `src/renderer/components/workspace/WorkspaceSidebar.tsx` — 侧栏底部新增主题快捷按钮
- `src/renderer/hooks/useWorkspaceController.ts` — 新增主题状态管理 hook
- `src/renderer/App.tsx` / `src/renderer/main.tsx` — 初始化主题（读取 localStorage 并设置 data-theme）
- `src/styles.css` 和 `src/renderer/styles/workspace.css` — 少量硬编码色值适配

## 任务拆解

- [x] P001: 新增深色主题设计 token（CSS 变量） <!-- scope: src/renderer/styles/layout.css -->
  - 在 `:root` 后新增 `[data-theme="dark"]` 选择器，覆盖所有颜色类 CSS 变量
  - 覆盖 `color-scheme` 为 `dark`
  - 覆盖所有语义色：`--brand-*`、`--bg-*`、`--text-*`、`--line*`、`--shadow-*`、`--focus-ring`
  - 覆盖状态色（success/warning/danger/info）为适配深色背景的版本
  - 覆盖 `--bg-elevated-glass` 为深色半透明背景
  - 滚动条颜色适配深色背景
  - 确保新增变量与浅色主题变量一一对应，无遗漏

- [x] P002: 实现主题状态管理与持久化 <!-- scope: src/renderer/hooks/useWorkspaceController.ts,src/renderer/main.tsx -->
  - 定义主题类型 `ThemeMode = 'light' | 'dark' | 'auto'`
  - 实现 `useTheme()` hook：读写 `localStorage` 键 `autoplan-theme`，解析系统偏好（`prefers-color-scheme`）
  - 在应用启动时（`main.tsx` 或 `App.tsx`）调用初始化逻辑，在 `<html>` 上设置 `data-theme` 属性
  - 监听 `matchMedia('(prefers-color-scheme: dark)')` 变化，当模式为 `auto` 时自动跟随切换
  - 导出 `useTheme()` 供设置页和侧栏使用

- [x] P003: 在设置视图新增"外观"配置卡片 <!-- scope: src/renderer/components/workspace/WorkspaceSettingsView.tsx -->
  - 在设置导航中新增"外观"标签项（排在"CLI 后端"或"循环控制"之后）
  - 外观卡片内容：使用 segmented control 提供"浅色 / 深色 / 跟随系统"三个选项
  - 切换时立即调用 theme hook 更新 `data-theme` 并持久化到 localStorage
  - 预览效果：切换时页面即时反映主题变化，无需刷新

- [x] P004: 在侧栏底部新增主题快捷切换按钮 <!-- scope: src/renderer/components/workspace/WorkspaceSidebar.tsx -->
  - 在 `sidebar-footer` 区域内新增主题切换图标按钮（月亮/太阳图标）
  - 点击循环切换 light → dark → auto → light
  - 增加 tooltip 或 title 显示当前主题模式
  - 与设置页"外观"卡片共享同一 theme hook，状态联动

- [x] P005: 适配硬编码色值与深色特殊处理 <!-- scope: src/styles.css,src/renderer/styles/workspace.css -->
  - 审查全量 CSS 中不在 CSS 变量范围内的硬编码色值（如 `.codex-log` 的 `#1b2438` 深蓝背景、`.search-locate-highlight` 动画中的 `rgba` 值）
  - 代码日志区域（`.codex-log`、`.code-wrap`、`.code-gutter`、`.log-body` 等）：深色主题下保持暗底亮字，确保在整体深色主题中仍有足够对比度
  - 阴影（`box-shadow`）在深色主题下调整为暗色阴影（黑色半透明而非蓝色半透明）
  - 渐变品牌色（`--brand-grad`）在深色主题下适当提亮
  - 验收视图的 `border-color` 硬编码色值（如 `.check` 的 `#c3cee0`）改为引用 CSS 变量

- [x] P006: 完整验收 <!-- scope: validation -->
  - 验收命令：`npm run build && npm run smoke`
  - 验收范围与通过标准：
    1. 构建成功，smoke 测试全部通过
    2. 手动切换主题为"深色"后，设置页、侧栏、需求/反馈列表、计划列表、任务列表、概览页、脚本页、验收页均显示深色配色，文字可读
    3. 切换为"跟随系统"后，更改操作系统主题设置，应用自动跟随
    4. 刷新应用或重启后，主题偏好保持不变（读取 localStorage）
    5. 三种模式之间切换无闪烁、无样式错乱

| A5 | 深色主题下代码日志区域可读 | 检查 `.codex-log` 和脚本代码编辑区 |
| A6 | 构建和 smoke 测试通过 | 执行 `npm run build && npm run smoke` |
| A7 | 浅色主题不受影响（回归） | 切换回浅色模式，所有页面与原版一致 |
