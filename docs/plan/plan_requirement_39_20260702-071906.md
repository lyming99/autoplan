# 需求 #39 开发计划：工作界面左侧面板支持拖拽调节尺寸

> 需求原文：工作界面左侧面板，应该支持拖拽调节尺寸；有个最小尺寸和最大尺寸。

## 需求解读

- 「工作界面左侧面板」即 `WorkspacePage` 中 `.workspace-shell` 网格的第一列 `.sidebar`（品牌区 + 项目切换 + 工作区/执行导航 + 主题/循环状态）。
- 「拖拽调节尺寸」：在侧栏与主区之间提供可拖拽的分隔手柄，沿水平方向调整侧栏宽度。
- 「最小尺寸和最大尺寸」：拖拽结果必须在区间内 clamp，无法超出；建议最小 `200px`、最大 `480px`，默认沿用当前固定值 `248px`。
- 非功能诉求：宽度应持久化（刷新/重开应用后保留）；窄屏堆叠布局（≤1080px）下不应出现可拖拽手柄；拖拽时禁止文本被选中、整页光标统一为 `col-resize`。

## 影响范围分析（基于源码）

- `src/renderer/pages/WorkspacePage.tsx`：`.workspace-shell` 的渲染处，侧栏与 `.workspace-main` 为同级兄弟节点，拖拽状态与手柄挂载点应在此处。
- `src/renderer/styles/workspace.css:1`：`.workspace-shell { display: grid; grid-template-columns: 248px 1fr; }` —— 第一列宽度需由固定 `248px` 改为 CSS 变量（如 `var(--sidebar-width, 248px)`）；并在 `@media (max-width: 1080px)` 堆叠断点内隐藏手柄。
- `src/renderer/hooks/`：新增一个拖拽 hook，与 `useTheme.tsx` / `useComposerDrafts.ts` 的 `localStorage` 持久化风格保持一致。
- `src/renderer/pages/WorkspacePage.test.tsx`：工作区 UI 功能的源码级回归测试集中地，拖拽相关断言应在此补充。

总体方案：第一列改用 CSS 变量 `--sidebar-width`（默认回退 `248px`），由新 hook 维护状态并经 `localStorage`（全局键 `autoplan.workspaceSidebarWidth`）持久化；在 `.workspace-shell` 内渲染一条绝对定位的垂直拖拽手柄（Pointer Events，兼顾鼠标/触摸），拖拽时在 `body` 上加 `data-sidebar-resizing` 以统一光标并禁用文本选中；手柄宽度命中区约 8px，双击复位默认宽度。

## 任务拆解

- [x] P001: 新增 useSidebarResize hook（拖拽交互 + 区间 clamp + 持久化） <!-- scope: src/renderer/hooks/useSidebarResize.ts -->
  - 验收要点：
    - 导出 `useSidebarResize()`，返回 `{ width, resizing, onPointerDown, onReset }`（命名可微调，但须同时给出宽度、拖拽中状态、手柄 pointerdown 回调、复位回调）。
    - 用 Pointer Events（`pointerdown` → `setPointerCapture` → `pointermove` → `pointerup`）实现拖拽，鼠标与触摸均可触发；组件卸载时清理监听、释放捕获。
    - 暴露纯函数 `clampSidebarWidth(value, min, max)`，对非有限值/越界值 clamp；常量 `SIDEBAR_MIN_WIDTH = 200`、`SIDEBAR_MAX_WIDTH = 480`、`SIDEBAR_DEFAULT_WIDTH = 248`。
    - 持久化键为 `autoplan.workspaceSidebarWidth`（与 `useTheme`/`useComposerDrafts` 的 `autoplan-*` 前缀风格一致），读写 `window.localStorage` 用 `try/catch` 包裹兼容不可用环境；首次加载读取失败时回退默认宽度。
    - 拖拽过程中以 `requestAnimationFrame` 节流写状态，避免高频刷新；`pointerup` 时最终值落库。
    - 双击手柄（`dblclick`）或调用 `onReset()` 时复位为默认宽度并落库。
