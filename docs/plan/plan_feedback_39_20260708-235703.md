# 移除 plan 进度区表格模板

## 需求概要

调整 AutoPlan 生成的最终 Markdown plan 格式，去除任务/进度区域中的预置表格，只保留规范的任务拆解 checkbox 行，以及由执行过程追加的进度记录和日志文件路径模式。重点保证 `## 任务拆解` 章节只包含顶层任务行与缩进验收要点，`## 进度区` 不再渲染任务状态表格头或预置表格行。

## 任务拆解

- [ ] P001: 调整结构化 PlanSpec 的 Markdown 渲染模板 <!-- scope: src/loop/planRenderer.js -->
  - 验收要点：渲染出的最终 Markdown 必须包含精确二级标题 `## 任务拆解`，所有开发任务只在该章节内以顶层 `- [ ] P001: 任务标题 ` 行呈现。
  - 验收要点：任务验收要点只能作为缩进普通 bullet 渲染，不能渲染为 checkbox、表格、引用块或代码块中的任务。
  - 验收要点：`## 进度区` 初始内容不包含 `\| 任务 \| 状态 \| 备注 \|`、表格分隔线或任何按任务预置的表格行，只保留后续执行追加 `- P001 AutoPlan 完成...；日志：docs/progress/logs/...` 的空间。
  - 验收要点：完整验收任务仍由 PlanSpec 规范化流程追加为连续编号的最后一个任务，scope 为 `validation`。
- [x] P002: 强化计划格式校验与任务解析约束 <!-- scope: src/loop/planGeneration.js,src/loop/planTaskSync.js,src/loop/planParser.js -->
  - 验收要点：格式校验要求最终 Markdown 具备精确的 `## 任务拆解` 章节，且任务行只能是该章节内的顶层 checkbox 行。
  - 验收要点：任务拆解区域中的表格行、引用块、代码块和嵌套 checkbox 不会被当成可执行任务解析入库。
  - 验收要点：缺少 scope 的任务行仍按既有规范补 ` `，完整验收任务使用 ` `。
  - 验收要点：历史或外部生成的带表格进度区内容不会影响任务解析，执行追加的进度日志 bullet 仍保留在 `## 进度区`。
- [x] P003: 同步计划生成提示词中的硬性格式要求 <!-- scope: src/loop/planGeneration.js,src/loop/builtinPlanGenerator.js -->
  - 验收要点：外部 CLI 和内置 LLM 的计划生成提示明确要求任务拆解只使用顶层 checkbox 行，不得使用普通段落、代码块、表格、引用块或嵌套 checkbox 表达任务。
  - 验收要点：提示词明确说明最终 Markdown 的 `## 进度区` 采用执行进度 bullet 加日志文件路径模式，不预置任务状态表格。
  - 验收要点：结构化 PlanSpec 路径继续只要求模型写 JSON 文件，任务编号、完整验收任务和 Markdown 渲染仍由 AutoPlan 确定性完成。
- [x] P004: 补充无表格进度区与任务格式回归测试 <!-- scope: src/loop/structuredPlanSpec.test.js,src/loop/planGeneration.test.js,src/loop/planTaskSync.test.js -->
  - 验收要点：覆盖 PlanSpec 渲染结果中 `## 进度区` 不含任务状态表头、表格分隔线和预置任务表格行。
  - 验收要点：覆盖 `## 任务拆解` 中只有顶层 `- [ ] P00N: ... ` 被解析为任务，表格、引用、代码块和嵌套 checkbox 均被忽略或判为不合规。
  - 验收要点：覆盖最终完整验收任务连续编号且 scope 为 `validation`，不会因为移除表格而丢失。
  - 验收要点：覆盖执行后追加的 `- P00N AutoPlan 完成...；日志：docs/progress/logs/...` 进度日志模式不会被误解析为新的开发任务。
- [x] P005: 完整验收 <!-- scope: validation -->

  - 验收要点：`## 进度区` 不再包含 Markdown 表格，仅保留执行过程追加的进度 bullet 与日志文件路径记录。
