# Reject duplicate MCP-submitted requirements

## 需求概要

Feedback #42 requires AutoPlan to identify and reject duplicate requirements sent through MCP. The current MCP create_requirement and create_feedback tools delegate to the shared IntakeService create paths, which always insert new rows. The plan is to add shared duplicate-intake detection before persistence, expose a clear MCP error response that points to the existing record, and cover the behavior with regression tests while preserving normal unique requirement and feedback creation.

## 任务拆解

  - 验收要点：createRequirement rejects a new requirement when the same project already has a non-closed requirement with equivalent normalized title and body.
  - 验收要点：createFeedback rejects a new feedback item when the same project and requirement association already has a non-closed feedback item with equivalent normalized title and body.
  - 验收要点：Duplicate comparison ignores insignificant whitespace differences such as leading/trailing spaces, repeated spaces, and CRLF versus LF line endings, but does not collapse genuinely different text.
