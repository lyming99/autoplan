因此本需求拆为四块工作：①产出新暗色主题设计稿（含色板与各组件暗色态预览）到 `ui/`；②把新 token 落地到 `layout.css` 的暗色 token 块；③重配 `workspace.css`/`styles.css` 的显式暗色组件覆盖；④让硬编码的深色代码/终端/活动日志面板在新暗色主题下对齐基调。最后统一验收。

## 新暗色主题设计方向（设计稿需落地）

- 总基调：从当前偏 GitHub 风格的纯冷灰黑（`#0d1117`）升级为更有品牌温度的**「深空蓝紫」(Midnight Indigo)** 暗色语言——背景层级用带极淡蓝紫调的中性深色，与品牌紫蓝渐变 `--brand-grad` 呼应，避免大面积纯灰导致的「死板/对比生硬」。
- 提升层级可读性：明确三档背景（app / elevated / soft）与 hover 的**递进亮度差**，并加深分隔线对比，使卡片在暗色下仍有清晰边界。
- 语义色按暗色专用调校：success/warning/danger/info 的**主色提亮到可在深底上保持 WCAG AA 对比度**，其 `*-bg` 用同色相极低饱和暗底（不喧宾夺主）。
- 一致化阴影：暗色阴影统一用更高不透明度的黑色软阴影（`rgba(0,0,0,.x)`），并规范 brand 渐变元素的投影色相。
- 代码/终端面板：在暗色主题下从「写死深蓝灰」对齐到新基调的更深一档背景 + 提亮前景，使代码区与页面融为一体而非突兀的色块。
