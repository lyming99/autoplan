# 需求 #32 开发计划：打开工作区默认显示需求界面

## 需求概述

需求 #32 要求用户打开项目工作区时，默认进入“需求”界面，而不是当前的“概览”或其它界面。该需求主要影响工作区路由参数解析、默认 Tab 状态、侧边栏高亮、主内容区展示，以及静态工作区原型的默认展示状态。

## 设计约束

- 默认进入工作区仅影响未指定 Tab 的打开场景，不应破坏通过 URL `tab` 参数进入指定模块的深链能力。
- “需求”作为默认 Tab 后，侧边栏高亮、页面标题、副标题和主内容区必须保持一致，不能出现高亮与内容不匹配。
- 用户手动切换到其它 Tab 后，应继续保持现有交互习惯和 URL 同步能力。
- 项目切换时应延续当前 Tab 的语义；若无显式 Tab，则落到新的默认“需求”界面。
- 静态 `ui/workspace.html` 原型如仍作为展示或回归参考，应同步默认态，避免实现与原型不一致。
- 完整验收节点只负责最终验收，不承载普通开发实现。

## 任务列表

- [x] P001: 统一默认工作区 Tab 规则 <!-- scope: src/renderer/pages/WorkspacePage.tsx,src/renderer/types.ts -->
  - 验收要点：
    - 工作区默认 Tab 具备明确的单一配置来源，默认值指向 `requirement`。
    - 未传入 URL `tab` 参数时，`WorkspacePage` 初始状态解析为“需求”界面。
    - 传入非法或未知 `tab` 参数时，回退到“需求”界面，不再回退到“概览”。
    - 传入合法 `tab` 参数时仍能进入对应模块，`overview`、`feedback`、`tasks`、`events`、`settings` 不受影响。

- [x] P002: 调整工作区导航与内容默认态 <!-- scope: src/renderer/pages/WorkspacePage.tsx -->
  - 验收要点：
    - 通过项目列表打开 `/projects/:id` 时，侧边栏“需求”项默认高亮。
    - 主内容区默认渲染需求列表和需求录入区域，概览区不再默认显示。
    - 页面标题与副标题默认展示需求模块对应文案，避免仍显示概览描述。
    - 用户切换到其它 Tab 后，侧边栏高亮、主内容区和标题文案同步更新。

- [x] P003: 兼容 URL 同步与项目切换 <!-- scope: src/renderer/pages/WorkspacePage.tsx -->
  - 验收要点：
    - 点击默认“需求”Tab 时，URL 参数处理符合默认 Tab 规则，不产生多余或误导性的 `tab` 状态。
    - 从需求界面切换到其它项目时，新项目仍默认打开需求界面或保留用户当前显式选择的合法 Tab。
    - 浏览器前进、后退或手动修改 `tab` 参数后，界面能按参数或默认规则正确刷新。
    - 搜索、计划阅读弹窗、循环状态刷新等现有工作区状态不因默认 Tab 调整出现异常重置。

- [x] P004: 同步静态工作区原型默认展示 <!-- scope: ui/workspace.html -->
  - 验收要点：
    - 静态原型侧边栏默认高亮“需求”，不再高亮“概览”。
    - 静态原型默认激活 `view-requirement` 内容区，`view-overview` 初始为非激活状态。
    - 静态原型顶部标题、说明文案和演示脚本初始状态与需求模块一致。
    - 手动点击其它导航项后，原型仍保持原有 Tab 切换行为。

  - 验收要点：
    - 最终验收命令：执行 `npm run check` 和 `npm run build`，两条命令退出码均为 0。
    - 验收范围：覆盖工作区默认 Tab 解析、URL 深链参数、侧边栏高亮、需求内容默认渲染、项目切换和静态原型默认态。
    - 手动验收：启动应用后从项目列表打开任一工作区，确认默认进入“需求”界面；再分别访问带 `?tab=overview`、`?tab=feedback`、`?tab=tasks` 的链接，确认仍进入指定界面。
    - 通过标准：无 TypeScript 或构建错误；默认打开工作区时需求界面可见且导航状态一致；合法深链与手动 Tab 切换无回归。

## 总体验收标准

- 功能正确性：打开项目工作区且未指定 Tab 时，默认展示“需求”界面，包含需求列表、需求录入区和需求模块标题说明。
- 深链兼容性：URL 中存在合法 `tab` 参数时优先按参数展示对应界面；参数缺失或非法时统一回退到“需求”。
- 交互一致性：侧边栏高亮、主内容区、页面标题、副标题和 URL 同步状态保持一致，不出现默认态错位。
- 原型一致性：`ui/workspace.html` 的默认高亮和默认内容区与实际 React 工作区保持一致。
- 最终验收命令：执行 `npm run check` 和 `npm run build`；两条命令均退出码为 0，并完成默认打开、合法深链、非法参数回退、项目切换和 Tab 手动切换的人工验证。
