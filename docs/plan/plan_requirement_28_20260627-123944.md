# 需求 #28 开发计划：Codex 增加 xhigh 思考深度

## 背景

当前系统已支持为 Codex CLI 配置思考深度，并在项目配置、需求/反馈录入、任务执行日志和前端展示中使用 `low`、`medium`、`high`。需求 #28 要求新增 `xhigh` 思考深度，目标是在不影响现有默认值和 Claude CLI 行为的前提下，让 Codex 任务可以选择并传递 `xhigh`。

## 设计约束

- `medium` 继续作为默认思考深度，历史数据缺省或非法值仍按 `medium` 处理。
- `xhigh` 仅对 Codex CLI 生效，Claude CLI 不显示、不保存为 Claude 专属参数，也不参与 Claude 命令拼装。
- 新增值应贯穿配置归一化、命令参数拼装、类型定义、表单选项、摘要展示和任务上下文读取链路。
- 不改变现有 `low`、`medium`、`high` 的语义、展示文案和已保存数据兼容性。
