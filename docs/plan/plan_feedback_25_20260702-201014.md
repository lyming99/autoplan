# 反馈 #25 开发计划：左侧导航对话高亮规则调整

## 背景

反馈要求左侧导航的「对话」模块标题不再高亮；只有具体会话项需要高亮，并且该高亮只应在聊天窗口打开状态下显示。

结合当前代码，左侧导航由 `src/renderer/components/workspace/WorkspaceSidebar.tsx` 渲染。「对话」分组标题按钮当前会在 `activeTab === 'chat'` 时追加 `active` 类，触发 `src/renderer/styles/workspace.css` 中 `.nav-item.active` 的高亮样式；会话行则通过 `conversation.id === chatState?.activeConversationId` 追加 `active` 类，未限制当前是否打开聊天窗口。因此需要把“分组标题展开态”和“具体会话选中态”拆开处理：标题只负责折叠/展开，不承担当前页高亮；会话项只有在 `activeTab === 'chat'` 且匹配当前会话时才显示选中高亮。

## 设计约束

- 不改变对话列表所属位置：会话导航继续由 `WorkspaceSidebar` 负责，`ChatView` 不重新引入旧的二级侧栏。
- 不改变点击入口语义：「对话」标题继续只切换折叠/展开；只有具体会话项打开聊天窗口并切换会话。
- 不改变新建、置顶、重命名、删除、加载更多等会话管理动作。
- 不扩大后端、IPC、数据库或聊天消息流范围，本次只处理左侧导航视觉状态与对应回归测试。
