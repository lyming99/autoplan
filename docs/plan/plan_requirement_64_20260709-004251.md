# 增加项目级 Prompt 设置并注入 AutoPlan 规范约束

## 需求概要

需求 #64 希望在项目维度增加可配置 Prompt，用于约束当前项目的代码规范、计划格式和执行偏好。结合现有代码，配置应随项目状态持久化并通过设置页编辑，随后在计划生成和任务执行提示中统一注入，同时保持现有 AutoPlan 任务拆解硬性格式、PlanSpec 结构化生成和最终 Markdown 渲染链路不被破坏。

## 任务拆解

- [x] P001: 新增项目级 Prompt 持久化字段与运行时配置 <!-- scope: src/database.js,src/loopService.js,src/loop/runtime.js -->
  - 验收要点：project_states 增加项目级 Prompt 字段，老数据库迁移后默认值为空字符串。
  - 验收要点：loop.configure 支持保存、清空和读取项目级 Prompt，未传入该字段时不意外覆盖已有配置。
  - 验收要点：不同项目的 Prompt 配置相互隔离，snapshot/status 返回当前项目自己的配置。
- [x] P002: 打通前端类型、表单和配置提交链路 <!-- scope: src/renderer/types.ts,src/renderer/utils/workspaceForms.ts,src/renderer/hooks/useWorkspaceController.ts,src/preload.js -->

  - 验收要点：从 snapshot 回填设置表单时保留已保存 Prompt，提交设置时将最新 Prompt 写回后端。
  - 验收要点：表单 dirty 判断包含 Prompt 字段，修改、清空 Prompt 都能触发保存状态变化。
- [x] P003: 在项目设置页提供 Prompt 编辑入口 <!-- scope: src/renderer/components/workspace/WorkspaceSettingsView.tsx,src/renderer/styles/workspace.css,src/renderer/styles/components.css -->
  - 验收要点：设置页新增项目级 Prompt 编辑区域，支持多行输入，适合填写代码规范、计划拆解偏好和执行约束。
  - 验收要点：Prompt 留空时表示不追加项目级约束，保存后再次进入项目仍能看到当前配置。
  - 验收要点：界面布局与现有设置页分组风格一致，在窄屏和桌面宽度下输入框、说明文字和保存按钮不重叠。
