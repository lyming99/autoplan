# 修复 OpenCode 附件模式把指针消息误解析为文件路径

## 需求概要

反馈 #9 的日志表明，反馈 #6 已成功把 Windows `.cmd/.bat` shim 下的多行结构化计划 prompt 完整写入临时附件，但实际调用仍在参数解析阶段失败：OpenCode 1.17.15 报错 `File not found: 你的完整唯一指令已作为附件...`。结合当前 `src/agentCli.js` 与本机 `opencode run --help` 可确认，`run [message..]` 是位置消息数组，`-f/--file` 是文件数组；现实现按 `-f <附件> <指针消息>` 排列，导致指针消息也被 `-f` 吞入文件列表。需要调整附件与位置消息的组装契约，并用能验证 OpenCode 参数归属而非只检查字符串顺序的回归场景，保证完整 prompt 能进入 autoplan-plan agent。
