# Plan: 首页项目路径支持链接模式打开

## 需求概述
需求 #32：`首页项目文件路径，要像workspace那样，支持链接模式打开`。

按当前代码判断，“首页”对应 `ProjectsPage` 的项目卡片列表；“workspace 那样”对应 `WorkspaceSidebar` 中工作区路径按钮使用 `project-path project-path-link mono`，点击后通过 `window.autoplan.openProjectFolder({ projectId })` 打开系统文件夹，并在无路径时禁用。首页当前已具备点击路径打开项目文件夹的行为，但仅使用 `project-path` 和内联 `pathLinkStyle` 重置按钮外观，没有复用 workspace 的链接样式钩子，因此用户在首页看到的路径仍不像链接。

## 现状分析

- `src/renderer/pages/ProjectsPage.tsx` 中项目卡片整体点击会进入工作区；路径区域是 `button.project-path`，点击时 `event.stopPropagation()` 后调用 `openFolder(project)`，避免误触发卡片导航。
- `src/renderer/components/workspace/WorkspaceSidebar.tsx` 中当前项目路径使用 `className="project-path project-path-link mono"`，无路径时禁用，点击调用 `openProjectFolder`。
- `src/renderer/styles/workspace.css` 已定义 `.project-switcher .project-path-link`，具备主题色、指针光标、悬停下划线、focus ring 和禁用置灰等链接模式样式，但作用域限制在 workspace 侧栏。
- `src/renderer/styles/layout.css` 定义首页通用 `.project-path`，当前是灰色等宽省略号文本；若首页要像 workspace 一样，需要将链接样式抽成可在项目卡片和 workspace 侧栏共同复用的样式，避免继续依赖内联样式。
- `src/renderer/pages/WorkspacePage.test.tsx` 已有源码级断言覆盖 workspace 路径链接样式，可追加首页项目路径链接模式的回归断言。

## 总体验收标准

- 命令验收：在仓库根目录执行 `npm run check && npm test && npm run smoke && npm run build`，所有命令必须成功完成且退出码为 0。
