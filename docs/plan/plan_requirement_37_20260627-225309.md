# 需求 #37 开发计划：计划阅读隐藏任务拆解并修复 Markdown 预览任务列表

## 背景

需求 #37 指出：“计划阅读前文界面，不要显示任务拆解，在markdown预览里面显示，但是markdown预览任务列表有问题，是不是markdown格式冲突？处理一下”。当前计划阅读弹窗同时展示了任务拆解解析摘要和 Markdown 正文预览，容易在阅读前文时重复暴露任务拆解内容；同时 Markdown 预览中的 GFM checkbox 任务行可能与 AutoPlan 任务拆解格式 `- [ ] P001: ... <!-- scope: ... -->`、HTML 注释 scope、以及全局任务列表样式发生渲染或样式冲突，需要统一处理。

## 设计约束

- 本计划只处理计划阅读全文弹窗、MarkdownReader 对任务列表的渲染兼容、相关样式隔离和必要测试覆盖，不改动计划生成、任务执行、并发调度、数据库结构或任务解析规则。
- “计划阅读前文界面”不再单独展示任务拆解解析结果或任务明细；任务拆解内容仅作为 Markdown 正文的一部分在预览区域展示。
- Markdown 预览必须兼容 AutoPlan 任务拆解硬性格式，不能因为 `- [ ]`、`P001:` 或 `<!-- scope: ... -->` 导致任务行丢失、合并、嵌套错位或 checkbox 可交互误导。
- MarkdownReader 的修复应保持通用 Markdown 阅读能力，避免影响普通列表、代码块、表格、引用块、链接和非任务 checkbox 的现有展示。
- 不要把运行测试、回归、构建拆成普通开发任务；最终检查统一放入最后的“完整验收”节点。

## 总体验收标准

- 阅读聚焦：计划阅读全文弹窗不再在 Markdown 正文预览前单独展示任务拆解解析摘要或任务明细，前文阅读不被任务拆解区域打断。
- 正文保留：计划文件中的 `## 任务拆解` 章节仍在 Markdown 预览中按原文结构展示，符合 AutoPlan 任务拆解硬性格式。
- 列表正确：标准任务行 `- [ ] P001: 任务标题 <!-- scope: ... -->` 独占一行、编号连续可见、标题完整、scope 注释隐藏且不破坏渲染。
- 样式隔离：Markdown 任务列表样式只作用于 `.markdown-reader`，不影响业务任务列表、计划列表和其他弹窗控件。
- 兼容稳定：普通 Markdown 内容、代码块中的任务格式文本、嵌套验收要点和主题配色均保持可读且无明显回退。
- 最终验收命令：执行 `npm run check && npm run smoke`；验收范围为计划阅读弹窗、MarkdownReader 任务列表渲染、样式隔离和现有 smoke 主流程；通过标准为命令退出码为 0，且手动验证需求 #37 描述的问题已修复。
