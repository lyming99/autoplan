# 需求 #48 开发计划：根据设计稿实现脚本模块

## 背景

需求 #48 要求"根据设计稿 `ui/scripts.html` 实现脚本模块"。设计稿定义了一个完整的"脚本模块"：用户可创建自定义脚本（Node.js / Bash / PowerShell / CMD 四种运行时），既能从弹窗中手动运行，也能挂到 AutoPlan 循环的各个阶段（钩子）自动触发。页面主体为脚本卡片宫格（带筛选、搜索、新建、启用开关、运行状态），点击卡片打开详情弹窗，弹窗内含代码编辑器（运行时切换 + 行号代码区）、运行日志（合并/stdout/stderr）、右侧配置面板（触发方式、挂载阶段、执行设置、上下文注入）。

当前 AutoPlan 渲染层工作区已有 概览/需求/反馈/任务与计划/事件流/设置 六个 Tab（`WorkspaceTab` 见 `src/renderer/types.ts`、导航见 `src/renderer/components/workspace/WorkspaceSidebar.tsx`），但没有任何"脚本"概念：数据库无脚本表，循环（`src/loopService.js`、`src/loop/*.js`）无钩子执行点，主进程/快照/类型/preload 均无脚本数据通路。本需求需端到端新增"脚本"实体：数据模型 → IPC/preload/类型/快照 → 循环钩子执行与手动运行 → 渲染层列表视图与详情弹窗 → 测试 → 完整验收。

## 设计约束

- **单运行时 + 单代码体模型**：设计稿弹窗里的 Node.js/Bash/PowerShell/CMD 四个 Tab 是"运行时选择器"——一个脚本记录只保存一种 `runtime` 与一段 `body` 代码。切换 Tab 即切换该脚本的 `runtime`（并相应改变展示的 `.node/.sh/.ps1/.bat` 扩展名与 EOL 提示），不为单个脚本存储四份代码。卡片上的 mono 路径（如 `scripts/pre-validation.node`）由脚本名 + 运行时扩展名派生或由用户填写，仅作展示标识，不强制落盘为真实文件。
- **代码体内联存储，不落盘**：脚本代码体保存在数据库 `scripts.body`，运行时由主进程直接喂给 `runShell`/解释器（或经临时文件），无需把脚本写成仓库内真实文件；卡片与弹窗的"路径"仅为显示用标识符。
- **复用既有 Icon 组件，禁止照搬设计稿 SVG sprite**：设计稿用 `<symbol>/<use>` sprite，但本应用图标体系是 `src/renderer/components/icons.tsx` 的内联 `<Icon name="..."/>` + `ICONS` 注册表。新增图标（script/code/bolt/terminal/play/stop/power/sliders/inject/clock/folder/history/copy/trash/search 等）必须按既有 24×24 stroke 风格在 `icons.tsx` 内新增常量并登记到 `ICONS`，不得引入 `<symbol>` sprite。
- **复用既有设计 token，禁止重复定义**：设计稿 `:root` 里的 `--brand-*`、`--bg-*`、`--text-*`、`--success/--danger/--info/--purple`、`--r-*`、`--shadow-*` 等已存在于 `src/renderer/styles/layout.css`。脚本模块的样式写入 `src/renderer/styles/workspace.css`，直接引用既有变量，不重复声明同名变量。
- **钩子执行复用 `runShell`，零默认影响**：钩子脚本通过 `loopService.js:1310` 的 `runShell(workspace, command, label, operation)` 执行（与验收命令同一通道），在 `workspaceToolEnv(workspace)` 之上叠加 `AUTOPLAN_*` 上下文环境变量。**默认无任何脚本时，循环行为必须与现状完全一致**——钩子是纯增量逻辑，仅当存在"已启用 + 匹配阶段"的脚本时才触发；任何单条钩子脚本的异常都必须被捕获并记为失败事件，绝不抛出导致循环崩溃或中断后续阶段（除非该脚本处于可中断阶段且 `fail_aborts` 开启并返回非零退出码）。
- **可中断语义受限**：仅 `validation:before`（以及未来可能的 `task:before`/`plan:before`）这类"前置"钩子在 `fail_aborts` 开启且退出码非零时中断当前阶段；`task:after`/`plan:after`/`loop:end`/`on:fail` 等后置/通知型钩子不中断流程，失败仅记事件。
- **安全模型**：脚本执行仍走主进程，渲染进程不直接接触 `child_process`；新增 IPC 遵循 `namespace:action` 命名（`scripts:create` 等）与既有 `requiredProjectId/requiredRecordId` 校验风格；增删改后返回 `loop.snapshot(projectId)`。
- **不回退既有能力**：现有项目/需求/反馈/计划/任务/验收/事件流/MCP/CLI 后端链路与导航均不得回退；scripts 仅新增 Tab 与通路。
- **文件长度约束**：`npm run check` 含文件长度检查。新增组件（列表视图、详情弹窗）若体积较大应拆分（如 `WorkspaceScriptsView.tsx` + `ScriptEditorModal.tsx`），避免单文件超限。
- **不要把运行测试/回归/构建拆成普通开发任务**；最终检查统一放入最后的"完整验收"节点。
