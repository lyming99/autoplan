# 需求 #44 开发计划：Claude CLI 在 plan 内保持会话上下文

## 背景

当前 Codex 后端会在同一个 plan 的串行任务之间复用 `codex_session_id`，失败重试/修复/验收事件也会带上会话信息；Claude 后端虽然通过 `claude --print` 非交互执行，但现有实现会清理 Codex 会话字段，导致同一 plan 内后续 Claude 任务无法继承前序分析上下文。本需求要求 Claude CLI 像 Codex 一样，在同一个 plan 内拥有可延续的会话上下文。

## 实施原则

- 保持 Codex 现有行为不回退，历史 `codex_session_id` 数据继续兼容。
- 会话语义升级为“Agent CLI 会话”，Claude 与 Codex 使用同一套 plan 内串行继承规则。
- 并发任务仍保持隔离，不自动复用同一 plan 的前序任务会话，避免上下文串扰。
- Claude 使用本机 CLI 支持的 `--session-id`/`--resume`/`--continue` 能力实现可恢复会话；具体参数由封装层集中生成。
