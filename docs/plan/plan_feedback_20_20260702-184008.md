# 反馈 #20 开发计划：AI 配置改为全局并刷新聊天状态

## 背景

反馈 #20 指出：AI 配置应是全局配置，不应按项目隔离；配置完成后，聊天窗口需要刷新配置状态。

基于当前代码分析，数据库初始化中已经出现 `migrateAiConfigsToGlobal()`，会把历史 `ai_configs.project_id` 置为 `NULL` 并做去重；但服务层、IPC、前端类型和设置页仍保留明显的项目级语义：

- `src/chat/aiConfigService.js` 的 `createAiConfig/listAiConfigs/resolveAiConfigForConversation` 仍按 `projectId` 创建、查询和回退。
- `src/main.js` 的 `ai-config:list/create` 仍要求 `projectId`，`chat:saveConfig` 只写旧 `settings.chat.*`，不更新 `ai_configs`。
- `src/preload.js`、`src/renderer/types.ts`、`src/renderer/utils/workspaceForms.ts` 仍把 `projectId` 作为 AI 配置 CRUD 的必要字段。
- `src/renderer/components/workspace/WorkspaceSettingsView.tsx` 的 AI 配置列表会随项目切换重置，并在缺少项目上下文时禁用新增。
- `src/renderer/hooks/useChat.ts` 只在项目/对话加载时拉取 AI 配置，设置页保存后没有面向聊天窗口的配置变更通知，导致 ChatView 的缺 key、provider/model、配置下拉等状态不能即时更新。

## 设计约束

- AI 配置表继续复用 `ai_configs`，以 `project_id IS NULL` 表示全局配置；历史项目级数据通过现有迁移提升为全局数据。
- 对话仍属于项目，历史消息仍按项目/对话隔离；本次只调整 AI 配置作用域，不改变聊天记录隔离模型。
- 对话仍可绑定某个 AI 配置 ID；绑定配置不存在或被删除时，降级到全局首个可用配置，再降级到内置默认值。
- API Key 仍只在主进程/数据库侧保留明文，渲染进程只接收 `hasApiKey` 与 `maskedKey`。
- 设置页保存、创建、编辑、删除 AI 配置后，聊天窗口必须无需切换项目或重启应用即可刷新配置状态。

  - 验收要点：`createAiConfig` 不再要求有效 `projectId`，新建配置写入 `project_id = NULL`，保留名称、provider、baseUrl、apiKey、model、temperature、thinkingDepth、thinkingBudgetTokens 的现有规范化和脱敏行为。
  - 验收要点：`listAiConfigs` 返回全局配置列表，查询条件以 `project_id IS NULL` 为准，不再按当前项目过滤。
  - 验收要点：`resolveAiConfigForConversation` 的回退链路调整为 `conversation.ai_config_id -> 全局首条配置 -> 内置默认值`，不再使用 `conversation.project_id` 查项目首条配置。
  - 验收要点：删除配置时继续清空所有引用该配置的 `conversations.ai_config_id`，不受项目限制。
  - 验收要点：`migrateAiConfigsToGlobal()` 保持幂等，历史项目级配置会提升为全局配置并去重，迁移后不会留下被服务层忽略的项目级默认配置。

- [x] P002: 调整主进程 IPC 与预加载 API 契约 <!-- scope: src/main.js,src/preload.js,src/renderer/types.ts,src/chat/chatController.js -->
  - 验收要点：`ai-config:list`、`ai-config:create` 不再要求或校验 `projectId`；预加载层和 `AutoplanApi` 类型同步移除 AI 配置 CRUD 载荷中的必填 `projectId`。
  - 验收要点：`chat:saveConfig` 不再只写旧 `settings.chat.*`，而是更新或创建全局默认 AI 配置；旧 `chat:getConfig` 如保留兼容，也应反映当前全局默认配置的脱敏状态。
  - 验收要点：`ai-config:create/update/delete` 与 `chat:saveConfig` 成功后，主进程向渲染进程广播统一的 AI 配置变更事件。
  - 验收要点：已有聊天 controller 的配置缓存不会在下一次发送时继续使用旧配置；可通过新增配置失效方法或在配置变更后清理非活跃 controller 实现。
  - 验收要点：旧调用方传入 `projectId` 时不报错但被忽略，避免破坏尚未同步的兼容路径。

- [x] P003: 设置页改为全局 AI 配置管理 <!-- scope: src/renderer/components/workspace/WorkspaceSettingsView.tsx,src/renderer/utils/workspaceForms.ts -->

  - 验收要点：新建配置按钮不再因缺少有效项目上下文而禁用；相关提示文案从“每个项目/当前项目”调整为“全局配置/所有项目共用”。
  - 验收要点：`aiConfigInputFromForm` 不再生成 `projectId` 字段，新增、编辑、删除均调用全局 AI 配置 API。
  - 验收要点：保存全局兼容配置或多配置表单后，设置页本身的列表、导航状态和 key 脱敏状态立即刷新。
  - 验收要点：编辑已有配置时，API Key 留空仍表示保留原密钥；显式填写新密钥会更新，删除配置会从列表中即时移除。

- [x] P004: 聊天窗口响应全局配置变更并刷新状态 <!-- scope: src/renderer/hooks/useChat.ts,src/renderer/components/workspace/ChatView.tsx,src/renderer/components/workspace/WorkspaceSidebar.tsx -->
  - 验收要点：`useChat` 改为加载全局 AI 配置列表，并订阅主进程的 AI 配置变更事件；收到事件后刷新 `aiConfigs` 和必要的对话绑定状态。
  - 验收要点：设置页配置好 API Key 后，聊天窗口的缺 key 空状态、输入框禁用状态、发送按钮状态、provider/model/思考深度下拉立即更新，无需重启、刷新页面或切换项目。

  - 验收要点：新建对话时使用全局首个有 API Key 的配置作为默认绑定，若没有可用密钥则保持未配置状态并显示现有缺 key 提示。
  - 验收要点：刷新配置状态不清空当前聊天记录、不打断非相关页面状态；若正处于生成中，当前生成可按既有流程结束，下一次发送使用最新配置。
