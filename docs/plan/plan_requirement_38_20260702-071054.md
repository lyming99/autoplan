# 需求 #38 开发计划：AI 配置界面简约化 + Dialog 编辑 + 遮罩防误关

## 需求概述

优化「AI 对话」设置面板里的 AI 配置项界面：

1. 列表采用简约的列表项风格（去掉冗余卡片层级）。
2. 新建 / 编辑配置改为通过 **Dialog（弹窗）** 完成，结构样式与「项目管理」弹窗一致。
3. 修改 Dialog 行为：**点击遮罩（边缘）不再关闭弹窗**，避免误操作丢失输入；只能通过显式的「关闭 / 取消」按钮关闭。

## 现状分析（基于源码）

- AI 配置 UI 集中在 `src/renderer/components/workspace/WorkspaceSettingsView.tsx` 的 `activePane === 'ai'` 分支（约 `573-826` 行）：
  - 列表：`ai-config-list` 容器 + `ai-config-item`（内含 `ai-config-info`（`ai-config-name`/`ai-config-meta`）与 `ai-config-actions`（编辑/删除按钮））。
  - 新建/编辑表单：当 `isAiConfigEditing`（`editingConfigId !== null`）时，在列表下方以**内联 `set-card`** 形式渲染（约 `644-824` 行），保存逻辑 `saveAiConfig()`（`222-259` 行）走 `aiConfigCreate`/`aiConfigUpdate`。
- 项目管理弹窗位于 `src/renderer/pages/ProjectsPage.tsx`（`253-343` 行），使用通用样式类：`.modal-mask` / `.modal` / `.modal-head`（标题 + `.modal-close`）/ `.modal-form` / `.modal-foot`，相关 CSS 在 `src/renderer/styles/components.css:431-441`。
- 「点击遮罩关闭」目前写在 `modal-mask` 的 `onClick` 上，出现在 `ProjectsPage.tsx:254`（创建/编辑）、`ProjectsPage.tsx:322`（删除确认）、`PlanList.tsx:211`、`PlanReaderModal.tsx:137`、`ScriptEditorModal.tsx:218`。
- `ai-config-item` 相关列表样式**当前未定义**（`workspace.css` 仅有 `.ai-config-error`，约 `697-708` 行）。

## 关键约束（不可破坏）

`WorkspacePage.test.tsx:241-245` 与 `WorkspacePage.projectId.test.js:80-109` 为**源码字面量断言**，要求以下内容必须继续出现在 `WorkspaceSettingsView.tsx` 源码中：

- `<div className="ai-config-error" role="alert">`（错误提示容器）
- `setAiConfigError(getErrorMessage(error, 'AI 配置保存失败'));`
- `await window.autoplan.aiConfigCreate(payload);`
- `setAiConfigError('配置名称不能为空');`
- `const startNewAiConfig = useCallback(() => {`、`onClick={startNewAiConfig}`、按钮文案 `新建配置`、上下文提示 `当前工作区缺少有效项目上下文，无法新建 AI 配置。`

→ **AI 配置表单的 JSX/逻辑必须保留在 `WorkspaceSettingsView.tsx` 内**，不得拆分到独立组件文件（否则上述断言失败）。

## 设计方案（最优工程方案）

抽出一个共享的 `Modal` 组件，统一封装弹窗结构（`.modal-mask`/`.modal`/`.modal-head`/`.modal-form`/`.modal-foot`）并**默认禁用遮罩点击关闭**。让项目管理弹窗与 AI 配置弹窗共同复用该组件，从而：

- 天然保证「样式与项目管理 Dialog 一致」（同一组件 → 同一样式，避免漂移）。
- 在组件层一次性落地「点击遮罩不关闭」，集中、不会漏改。

为满足上述源码断言约束，AI 配置弹窗**仅复用 `Modal` 外壳**，其内部表单字段 JSX 与保存逻辑仍写在 `WorkspaceSettingsView.tsx` 中作为 `Modal` 的 children。

> 说明：本轮仅处理需求明确提及的「表单/配置类」弹窗（项目管理创建/编辑、删除确认、AI 配置新建/编辑）。`PlanReaderModal`、`ScriptEditorModal`、`PlanList` 的并行确认弹窗属于阅读/查看类弹窗，保持其现有遮罩关闭行为，不在本需求范围内（如需统一可在后续需求处理）。
