# 让任务 scope 可打开文件真正支持点击打开

## 需求概要

任务列表中 scope 文件已根据后端解析结果显示“可打开”语义，但当前只是静态 chip，无法触发已有的工作区文件打开能力。需要在任务列表中把 canOpen 的 scope 渲染为可点击的链接式控件，并通过现有 workspace:openFile IPC 按用户配置的软件或命令打开文件；unknown、validation、不可打开或不存在的 scope 继续保持只读状态。

## 任务拆解

- [x] P001: 接通任务 scope 文件打开回调 <!-- scope: src/renderer/types.ts,src/renderer/hooks/useWorkspaceController.ts,src/renderer/pages/WorkspacePage.tsx -->
  - 验收要点：渲染进程类型声明包含 window.autoplan.openWorkspaceFile，入参支持 projectId、filePath、mode、command，返回 ok/error 结果。
  - 验收要点：Workspace 控制器提供打开 scope 文件的回调，使用当前 projectId、scope 路径以及 scopeFileOpenSettings 中配置的打开方式和命令调用 openWorkspaceFile。
  - 验收要点：打开失败时向用户展示后端返回的错误信息，成功时不破坏当前任务列表筛选、计划选择和 Plan 阅读器状态。
  - 验收要点：WorkspacePage 将打开 scope 文件回调传入任务列表，任务列表不直接依赖全局 window.autoplan。
- [x] P002: 将可打开 scope chip 渲染为链接式控件 <!-- scope: src/renderer/components/PlanLists.tsx,src/renderer/components/plans/TaskList.tsx -->
  - 验收要点：file.canOpen 为 true 的 scope 文件显示为可点击的链接式按钮，并保留文件路径、状态文案、title 和键盘可访问性。
  - 验收要点：点击可打开 scope 时调用父级 onOpenScopeFile 回调并传入对应文件路径，不触发任务执行、停止、分组折叠或计划预览等其他动作。
  - 验收要点：unknown、validation、目录、缺失文件和 canOpen 为 false 的 scope 仍以只读 chip 展示，不出现可点击交互。
  - 验收要点：任务列表的计划文件预览按钮和任务执行/停止按钮行为保持不变。
- [x] P003: 完善可点击 scope 的视觉与可访问状态 <!-- scope: src/renderer/styles/components.css -->
  - 验收要点：可点击 scope chip 与现有 openable 视觉语义一致，同时具备链接式 hover、focus-visible 和 pointer 光标反馈。
  - 验收要点：按钮默认样式不会破坏 chip 的尺寸、换行、文本省略和小屏布局。
  - 验收要点：不可点击 scope chip 的 warning/info/success 状态样式保持现状，不产生误导性的交互反馈。
- [x] P004: 补充任务 scope 打开链路回归覆盖 <!-- scope: src/renderer/pages/WorkspacePage.test.tsx,src/renderer/utils/planTasks.test.ts -->
  - 验收要点：覆盖 WorkspacePage 将 onOpenScopeFile 传给 TaskList，并确认控制器调用 openWorkspaceFile 时携带当前项目、scope 路径和用户配置的打开方式。
  - 验收要点：覆盖 TaskList 对 canOpen scope 渲染为 button/link 控件，对 unknown、validation 和不可打开 scope 保持只读 chip。
  - 验收要点：覆盖 scopeFileStatus、scopeFileClassName 等现有语义不变，避免“可打开”状态显示和实际交互再次脱节。
- [x] P005: 完整验收 <!-- scope: validation -->
  - 验收要点：最终验收命令为 npm run build && npm test
  - 验收要点：TypeScript 编译和 Vite 构建通过，新增 openWorkspaceFile 类型与 TaskList 回调没有类型错误。
  - 验收要点：现有 Node 测试与新增回归测试全部通过。
  - 验收要点：任务列表中 canOpen scope 文件可点击，并按设置的 system、folder、vscode 或 command 打开方式调用后端打开文件。
  - 验收要点：unknown、validation、缺失或不可打开 scope 不可点击，样式和提示语仍能准确表达状态。
