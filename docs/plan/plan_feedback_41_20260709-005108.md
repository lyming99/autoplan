# Fix task completion timestamp format

## 需求概要

Feedback #41 reports that the task completion note appended to the plan progress section currently uses an ISO timestamp such as 2026-07-07T14:06:06.811Z, but the expected display format is yyyy-MM-dd HH:mm:ss. The affected write path is markTaskCompletedInPlan in src/loop/taskExecution.js. Internal database and event timestamps can keep the existing ISO format; only the Markdown progress note should use the user-facing second-level date-time format.
