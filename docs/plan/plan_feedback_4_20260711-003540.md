# 修复 OpenCode CLI 计划生成因权限参数失效而退出

## 需求概要

反馈 #4 的根因位于 OpenCode CLI 参数适配：当前代码在所有 `opencode run` 调用中固定传入 `--dangerously-skip-permissions`，但本机 OpenCode 1.17.15 的 `run --help` 及当前官方 CLI 契约已改为使用 `--auto` 自动批准未被显式拒绝的权限请求，旧参数会在模型执行和 PlanSpec 文件写入前被命令行解析器拒绝，因此计划无法生成。现有单元测试仅检查 `run`、`--format`、输出与 prompt 投递方式，未断言 OpenCode 权限参数，导致该兼容性回归未被发现。本次修复聚焦 `src/agentCli.js` 的 OpenCode 参数构造，并补充精确的参数回归单元测试。

## 任务拆解

- [x] P001: 更新 OpenCode CLI 无人值守权限参数 <!-- scope: src/agentCli.js -->
  - 验收要点：OpenCode 的非交互 `run` 调用使用当前 CLI 支持的 `--auto`，不再传入会导致 OpenCode 1.17.15 参数解析失败的 `--dangerously-skip-permissions`。
  - 验收要点：`--auto` 与 `--format`、`--session`、`--title`、`--agent` 及位置 prompt 的参数顺序保持合法，结构化计划生成能够进入 agent 执行并写入 PlanSpec JSON。
  - 验收要点：OpenCode 任务执行、会话恢复和普通计划生成共用的参数构造行为同步修复，Codex、Claude 与 Oh My Pi 的既有参数不受影响。
