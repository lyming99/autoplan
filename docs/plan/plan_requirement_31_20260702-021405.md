# Plan: 修复 AI 配置无法创建

## 需求概述

需求 #31：AI 配置无法创建。

目标是在不回退需求 #28 多 AI 配置能力的前提下，修复设置页新建 AI 配置失败的问题，并补齐创建链路的迁移兼容、错误提示和回归测试。按当前代码判断，影响链路集中在 `ai_configs` 数据表、`src/chat/aiConfigService.js` CRUD、`main.js` 的 `ai-config:*` IPC、`preload.js` 暴露接口，以及 `WorkspaceSettingsView.tsx` 的 AI 配置表单。

## 现状分析

- `src/database.js` 已在 `CREATE TABLE IF NOT EXISTS ai_configs` 中声明 `thinking_depth`、`thinking_budget_tokens`，但对已存在的旧 `ai_configs` 表没有 `ensureColumn` 迁移；旧库升级后执行 `createAiConfig` 插入这两个字段时可能直接失败。
- `src/chat/aiConfigService.js` 的 `createAiConfig` 会写入 `thinking_depth`、`thinking_budget_tokens` 并返回脱敏配置，适合作为修复入口统一处理字段归一化和旧库兼容。
- `src/main.js` 已注册 `ai-config:create`，`src/preload.js` 已暴露 `aiConfigCreate`，`src/renderer/types.ts` 已定义 `AiConfigCreateInput`；创建失败时主要需要保证错误边界清晰、前端能展示原因。
- `src/renderer/components/workspace/WorkspaceSettingsView.tsx` 当前 `saveAiConfig` 的 `catch` 静默吞掉异常，用户看到的就是“无法创建”且没有可操作信息。
- 当前测试覆盖了 `llmClient` 和 `chatController`，但没有专门覆盖 `aiConfigService` 创建、旧库迁移和设置页创建交互的回归断言。

## 任务拆解

- [x] P001: 补齐 AI 配置表迁移与创建入库兼容 <!-- scope: src/database.js,src/chat/aiConfigService.js -->
  - 验收要点：`AppDatabase.migrate()` 对既有 `ai_configs` 表幂等补齐 `thinking_depth TEXT`、`thinking_budget_tokens INTEGER`，旧版本数据库升级后无需重建库即可创建新配置。
  - 验收要点：`createAiConfig` 对 `projectId`、`name`、`provider`、`baseUrl`、`apiKey`、`model`、`temperature`、`thinkingDepth`、`thinkingBudgetTokens` 做统一归一化；非法 provider 回退到 `openai`，空名称仍拒绝保存。
  - 验收要点：创建成功返回脱敏后的 `AiConfig`，不泄露原始 `api_key`，`hasApiKey` 与 `maskedKey` 和实际保存值一致。
  - 验收要点：保留 `migrateChatToAiConfigs` 的默认配置迁移语义，不重复生成默认配置，不影响已有 `list/get/update/delete/resolve` 行为。

  - 验收要点：`ai-config:create` 仅接收并透传创建所需字段，`projectId` 继续走 `requiredProjectId` 校验，避免无项目上下文写入。
  - 验收要点：创建、更新、删除、获取配置的 IPC 返回结构与 `AutoplanApi` 类型保持一致；创建成功可立即被 `aiConfigList({ projectId })` 查询到。
  - 验收要点：后端校验失败或数据库失败时保留明确错误信息，渲染层可拿到 `Error.message`，不再表现为静默失败。
  - 验收要点：既有 `chat:saveConfig`、`chat:getConfig`、`conversation:*` IPC 不受影响。

- [x] P003: 修复设置页新建配置交互与失败提示 <!-- scope: src/renderer/components/workspace/WorkspaceSettingsView.tsx,src/renderer/utils/workspaceForms.ts,src/renderer/styles/workspace.css -->

  - 验收要点：创建模式下空 API Key 按空串保存；编辑模式下 API Key 留空继续保留原密钥，不误清空。

## 总体验收标准

- 命令验收：在仓库根目录执行 `npm run check && npm test && npm run smoke && npm run build`，所有命令必须通过。
- 功能验收：在任一项目的“设置 > AI 对话”中创建 AI 配置，保存后配置列表立即刷新；重启应用后配置仍可查询；创建失败时显示明确错误而不是静默无响应。
- 数据验收：新库和旧库都具备完整 `ai_configs` schema；旧库迁移不会丢失已有配置或重复生成默认配置。
- 安全验收：渲染层和 IPC 返回值不包含原始 API Key，仅返回 `hasApiKey` 和 `maskedKey`。
