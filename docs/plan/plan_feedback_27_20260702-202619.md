# 反馈 #27 开发计划：左侧面板工作区/执行分组改为独立滚动视图

> 反馈原文：左侧面板，优化一下布局，只是将工作区、执行放到滚动视图。

## 背景 / 根因分析

工作区左侧面板 `.sidebar`（`src/renderer/styles/workspace.css:3`）当前为：

```
.sidebar { display: flex; flex-direction: column; gap: 6px; height: 100vh; overflow-y: auto; }
```

即**整个侧边栏作为一个滚动容器**。其内部从上到下依次为（`src/renderer/components/workspace/WorkspaceSidebar.tsx:207-457`）：

1. `.brand`（AutoPlan 标识）
2. `.back-link`（返回项目列表）
3. `.project-switcher`（当前项目选择 + 工作区路径）
4. `NAV_GROUPS.map(...)` —— 即「工作区」分组（概览/需求/反馈/验收/对话）与「执行」分组（计划与任务/脚本/事件流/设置）的导航项，其中「对话」还包含可展开的会话子列表
5. `.sidebar-footer`（主题切换 + 循环运行状态/配置摘要，靠 `margin-top: auto` 吸附底部）

由于 `.sidebar` 整体 `overflow-y: auto`，当导航项（尤其是「对话」会话列表展开后）高度超出视口时，**连同顶部的品牌/返回/项目切换、底部的主题/循环摘要一起被卷动**，导致常用操作（切换项目、返回列表、看循环状态）需要上下滚动才能触及，体验不佳。

反馈明确诉求是「**只是**将工作区、执行放到滚动视图」—— 即一个最小、定向的布局调整：把「工作区」「执行」两组导航项收敛进一个**独立的内部滚动区域**，而顶部（品牌/返回/项目切换）与底部（主题/循环摘要）固定不动。这是典型的「固定头部 + 可滚动主体 + 固定底部」侧栏布局。

工程方案：
- **TSX**：在 `WorkspaceSidebar.tsx` 中将渲染 `NAV_GROUPS` 的那段（当前为 `.sidebar` 的直接子节点 `NAV_GROUPS.map(...)`）包进一个滚动容器元素 `<nav className="sidebar-nav" aria-label="工作区导航">…</nav>`，使其成为 `.sidebar` 中位于 `.project-switcher` 与 `.sidebar-footer` 之间的独立区块；不改动任何导航项的渲染逻辑与交互。
- **CSS**：将 `.sidebar` 由 `overflow-y: auto` 改为 `overflow: hidden`（侧边栏本身不再滚动，仅作为 flex 列容器）；新增 `.sidebar-nav { flex: 1 1 auto; min-height: 0; overflow-y: auto; overflow-x: hidden; }` 作为真正的滚动区。其中 `min-height: 0` 是关键——flex 子项默认 `min-height: auto` 会阻止收缩，缺了它内容溢出时不会触发滚动而是撑破容器。
- **固定底部**：`.sidebar-footer` 既有的 `margin-top: auto`（workspace.css:264）在 `.sidebar-nav` 设为 `flex: 1 1 auto` 后仍会吸附底部，需确认其在导航项较少/较多时都稳定贴底、不随导航滚动。
- **响应式**：`@media (max-width: 1080px)`（workspace.css:510-513）将 `.sidebar` 改为 `height: auto` 的堆叠布局，此时 `.sidebar-nav` 的内部滚动不再适用，需在该断点显式置为 `overflow: visible; max-height: none;`，保证窄屏下分组全部平铺、由页面整体滚动。

关键风险点：`.sidebar-nav` 改为 `overflow: hidden/auto` 后，其内部「对话」会话项的操作菜单 `.nav-sub-item__menu`（`position: absolute`，向下展开）若靠近可视区底部可能被滚动容器裁剪——这与改动前（菜单本就在滚动的 `.sidebar` 内）行为一致，需在验收中确认未引入额外裁剪回归。

- [x] P002: 调整侧边栏 CSS 使导航分组成为独立滚动区、顶/底固定 <!-- scope: src/renderer/styles/workspace.css -->
  验收要点：
  - `.sidebar`（workspace.css:3）由 `overflow-y: auto` 改为 `overflow: hidden`，保留 `display: flex; flex-direction: column; gap: 6px; padding; height: 100vh; background; border-right` 等既有声明不变——侧边栏本身不再整体滚动。
  - 新增 `.sidebar-nav { flex: 1 1 auto; min-height: 0; overflow-y: auto; overflow-x: hidden; }`，使其成为可纵向滚动的主体区；`min-height: 0` 必须存在以允许 flex 子项收缩触发滚动。
  - `.sidebar-footer`（workspace.css:264，`margin-top: auto`）在导航项较少或较多时均稳定贴底，不随导航内容滚动；导航项不足一屏时不出现多余空白导致 footer 上浮（依赖 `.sidebar-nav` 的 `flex: 1 1 auto` 占据剩余空间）。
  - `@media (max-width: 1080px)` 断点内补 `.sidebar-nav { overflow: visible; max-height: none; }`（或等效），与该断点既有的 `.sidebar { height: auto; max-height: none; }`（workspace.css:513）配合，保证窄屏堆叠时分组全部平铺、不出现内部滚动条、由页面整体滚动。
  - 滚动条外观与既有侧边栏一致（浅/深主题下均不出现样式回归），`overflow-x: hidden` 不裁剪导航项文本（标题仍走既有省略号规则）。
  - 仅新增/调整上述规则，不改动 `.nav-item`、`.nav-subgroup`、`.nav-sub-item` 等其它既有样式。
