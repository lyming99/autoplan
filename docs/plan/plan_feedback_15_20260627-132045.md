# 反馈 #15 开发计划：修复阅读全文看不到任务拆解

## 背景

反馈 #15 指出：“阅读全文没有办法看到任务拆解，是否解析存在问题？” 当前风险点集中在两类链路：一是计划文件中的任务行没有被 `syncPlanTasks` 正确解析并同步到 `plan_tasks`，导致计划列表、任务列表和并发建议为空；二是“阅读全文”弹窗只渲染 Markdown 原文，缺少解析结果摘要、异常提示和任务拆解兜底展示，用户无法判断是计划内容缺失、任务格式不合规，还是解析器失败。

## 设计约束

- 本计划只处理 Plan 全文阅读器、任务拆解解析、任务同步和相关提示，不改动任务执行、CLI 调用、发布打包等无关流程。
- 修复优先保证固定格式任务行 `- [ ] P001: 任务标题 <!-- scope: ... -->` 被稳定解析；兼容格式只作为兜底，不降低新计划生成的严格格式要求。
- “阅读全文”必须让用户能看到任务拆解状态：解析成功时展示任务摘要，解析失败或任务数为 0 时展示明确原因和排查提示。
- 不要把运行测试、回归、构建拆成普通开发任务；最终检查统一放入最后的“完整验收”节点。

## 总体验收标准

- 解析可靠：标准格式任务行全部被同步到 `plan_tasks`，任务编号、标题、状态、排序和 scope 与 Plan 原文一致。
- 异常可见：当任务拆解无法被解析时，事件或“阅读全文”弹窗能明确提示解析异常，不再让用户误以为没有任务拆解。
- 阅读完整：“阅读全文”弹窗既能展示完整 Markdown 正文，也能展示解析后的任务拆解摘要；用户可以直接判断任务拆解是否存在和是否可执行。
- 格式约束一致：计划生成提示、README 示例和解析器规则保持一致，继续要求固定任务格式与必填 scope。
- 无关功能不回退：任务执行、并发建议、scope 文件打开、计划状态统计和已有 Markdown 渲染能力不因本次修复退化。
- 最终验收命令：执行 `npm run check && npm run smoke`；验收范围为 Plan 任务解析、任务同步、阅读全文弹窗展示、异常提示和现有 smoke 主流程；通过标准为命令退出码为 0，且手动验证标准 Plan 与异常 Plan 的阅读表现均符合预期。
