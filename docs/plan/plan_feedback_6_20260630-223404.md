# 开发计划：已完成任务支持重新执行与补充内容

## 背景

反馈 #6 指出：已完成的任务需要支持"重新执行"或"补充内容"。当前状态：

1. **TaskList 的"执行"按钮对已完成任务禁用**：`TaskList.tsx` 第 159 行 `disabled={completed || running}` 阻止已完成任务再次执行
2. **后端 `runTask()` 已支持重新执行**：`startTaskRun()` 会将已完成任务状态重置为 `running`、清空 `finished_at`、更新 `started_at`，但缺少 plan 文件旧完成标记的清理
3. **已完成计划无法追加任务后自动执行**：`IntakePanel` 的"+ 追加任务"按钮对已完成计划仍显示，但追加后 plan 仍保持 `completed` 状态，新任务不会被 loop 拾取执行
4. **已完成任务的 plan 文件残留**：重新执行时 plan 文件中的 `[x]` checkbox 和"AutoPlan 完成"注释不会在启动时清理，仅会在再次完成时覆盖

## 涉及文件

- `src/renderer/components/plans/TaskList.tsx` — 启用已完成任务的"重新执行"按钮
- `src/loopService.js` — 重新执行时清理 plan 文件旧标记；支持追加任务后重新激活已完成计划
- `src/loop/taskExecution.js` — 重新执行前清理 plan 文件中的 checkbox 和完成注释

## 总体验收标准

| 维度 | 标准 |
|------|------|
| 已完成任务重新执行 | 点击"重新执行"按钮后任务从 completed 状态进入 running、正常调用 CLI 后端、完成后正确写回 plan 文件 |
| Plan 文件清理 | 重新执行时旧 checkbox 重置为 `[ ]`、旧完成注释移除；再次完成后 checkbox 正确变 `[x]`、只有一条完成注释 |
| 已完成计划追加任务 | 追加新任务后 plan 从 completed 变为 running、新任务可被 loop 拾取执行 |
| 不破坏现有行为 | pending/running 任务的行为不变；非完成计划追加任务的行为不变 |
| 静态检查 | `npm run check` 零错误 |
| 冒烟测试 | `npm run smoke` 全部通过 |
