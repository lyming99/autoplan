# 反馈 #18 开发计划：移除搜索框旁可搜索数量

## 背景

反馈 #18 指出：“搜索框旁边不需要显示可搜索数量”。当前工作区搜索栏在未输入关键字时，会在搜索框旁展示“可搜索 N 条”；输入关键字后同一位置会展示“命中 N 条”。该反馈仅要求移除搜索框旁边的可搜索总数展示，避免界面信息冗余，但不应影响搜索能力、搜索结果统计或搜索结果概览中的命中数量展示。

## 设计约束

- 本计划只调整搜索框旁边的辅助数量文案，不改变统一搜索索引、搜索结果计算、筛选逻辑和列表过滤行为。
- 未输入搜索关键字时，搜索框旁不再展示“可搜索 N 条”或类似总量提示。
- 已输入搜索关键字时，可以继续保留“命中 N 条”反馈，用于提示当前查询结果数量；如实现选择完全移除旁侧数量，也必须确保搜索结果概览仍显示命中数量。
- `SearchResults` 组件中的搜索结果总数、来源分组数量和分组内数量不属于“搜索框旁边”的展示，应保持现有能力，避免误删结果统计。
- 不要把运行测试、回归、构建拆成普通开发任务；最终检查统一放入最后的“完整验收”节点。

## 总体验收标准

- 展示简化：搜索框为空时，搜索框旁边不再展示“可搜索 N 条”或任何等价的可搜索总数文案。
- 搜索可用：输入关键字后，统一搜索仍能命中需求、反馈、任务、Plan 和事件，列表过滤、结果跳转和清空搜索行为保持正常。
- 统计保留：搜索结果概览中的总命中数、来源分组数量和分组内数量保持可用，不因移除搜索框旁可搜索数量而被误删。
- 空态清晰：无搜索结果时仍有明确提示和清空搜索入口；清空后恢复完整工作区列表且不显示可搜索总量。
- 影响可控：项目页搜索框、工作区列表统计、任务筛选数量、事件展示和计划/反馈/需求主流程不受本次 UI 文案调整影响。
- 最终验收命令：执行 `npm run check && npm run smoke`；验收范围为工作区搜索栏展示、统一搜索结果概览、搜索过滤与跳转、无结果空态和现有 smoke 主流程；通过标准为命令退出码为 0，且手动验证搜索框空态旁侧无可搜索数量展示。
