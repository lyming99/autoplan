# 反馈 #15 开发计划：验收模块同时展示「待验收」与「已完成验收」

## 背景与目标

反馈指出：验收模块要**同时显示**「待验收」功能与「已完成验收」功能。

基于当前代码梳理的现状与根因：

1. **「已完成验收」目前是次要的、被截断的「最近」面板。** `src/renderer/components/workspace/AcceptanceView.tsx` 顶部为「待验收」一级区块（`accept-section-head` + `accept-groups`，按计划分组的 `PendingPlanCard`/`PendingTaskRow`，带进度条、文件路径、scope，信息丰富）；而已验收项仅作为底部可折叠的 `AcceptedSection`（标题「已验收（最近）」）以**扁平列表**展示，视觉权重低、易被忽略。
2. **取数被硬性截断在 50 条。** `src/renderer/utils/planTasks.ts` 的 `buildRecentAccepted(plans, tasks, limit = 50)` 末尾 `.slice(0, limit)`，导致已验收历史超过 50 条后旧记录被静默隐藏——「已完成验收」并不完整，与「待验收」一级区块不对等。
3. **「已完成验收」未按计划分组，无法与「待验收」对齐阅读。** `buildRecentAccepted` 返回扁平 `AcceptedRecord[]`（计划与任务混排），不像 `buildAcceptanceGroups` 那样按计划聚合；用户难以像审视待验收项那样结构化地复核「哪些计划的哪些任务已经验收」。
4. **顶部汇总只体现待验收一侧。** `.accept-bar` 的 `accept-meta` 仅展示「待验收 N 个计划 / M 个任务 / 最近验收时间」，没有同时给出「已完成验收」的计数，无法一眼同时看到两类功能的状态。

目标：把「已完成验收」从被截断的次要「最近」面板提升为与「待验收」对等的一级区块——**两个区块始终同时可见**；已完成验收展示**全部**已验收项并按计划分组；顶部汇总同时呈现两侧计数；保留既有逐项/批量验收与取消验收、多选、折叠等全部交互。验收态与执行态正交的既有语义（`accepted_at` 非空=已验收、`NULL`=未验收，仅可对「已完成」项验收）保持不变，不新增执行态取值、不改 IPC。
