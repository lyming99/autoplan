# 需求 #30 开发计划：优化项目架构，避免超长代码

## 背景

当前项目已形成 Electron 主进程、循环执行服务、渲染端工作区页面和计划列表组件等核心模块，但部分文件承担了过多职责：`src/loopService.js` 超过 3000 行，`src/renderer/pages/WorkspacePage.tsx` 超过 1400 行，`src/renderer/components/PlanLists.tsx` 接近 1000 行。需求 #30 要求“项目架构优化一下，避免超长代码”，目标是在不改变现有功能行为的前提下，将大文件按职责拆分为更清晰的服务、工具、Hook 和组件，并增加文件长度巡检能力，降低后续维护成本。

## 设计约束

- 本次优化以等价重构为主，不新增业务功能，不改变现有数据结构、IPC 契约、计划执行语义和 UI 操作路径。
- 拆分后的模块应有明确职责边界：循环服务负责编排，计划生成、任务执行、并发分析、快照映射、附件提示和 CLI 配置归一化应下沉到独立模块。
- 前端页面应拆分为页面容器、状态 Hook、搜索过滤工具、侧边栏、设置视图、概览视图和列表组件，避免单个页面文件继续承载全部逻辑。
- 重构后应保留既有导出 API 或提供兼容导出，避免一次性影响过多调用方。
- 文件长度巡检应排除 `dist`、`release`、`node_modules`、日志和生成物，仅约束可维护源码与脚本。
- 完整验收节点只负责最终验收，不参与普通开发任务并发或功能实现。

- 最终验收命令：`npm run check`、`npm run build`、`npm run smoke`。
- 验收范围：循环执行服务、计划生成、任务执行、并发分析、快照映射、工作区页面、计划/任务/事件列表、全局样式拆分和文件长度巡检。
