# 需求 #46 开发计划：创建项目支持文件夹选择/输入与路径链接打开系统文件夹

## 背景

当前创建/编辑项目时，工作区路径只能在文本输入框中手动填写（`src/renderer/pages/ProjectsPage.tsx` 创建/编辑弹窗的“工作区路径”字段），缺少通过系统文件夹选择器定位目录的能力。同时，项目工作区路径在两处都以纯文本展示：项目卡片（`src/renderer/pages/ProjectsPage.tsx` 的 `.project-path`）与工作区侧边栏（`src/renderer/components/workspace/WorkspaceSidebar.tsx` 的 `.project-path`），均无法点击在系统文件管理器中打开对应文件夹。本需求要求：创建项目时同时支持“选择文件夹”和“输入文件夹”两种方式；项目路径展示位置支持以链接形式打开系统文件夹。

## 实施原则

- 保留现有“手动输入文件夹路径”的文本输入，在其基础上新增“选择文件夹”入口，两种方式并存、互不影响。
- 文件夹选择与打开均通过 Electron 主进程完成（`dialog.showOpenDialog` 与 `shell.openPath`），经 `src/preload.js` 的 contextBridge 暴露，渲染进程不直接接触 Node API，符合现有 contextIsolation 安全模型。
- “打开系统文件夹”对齐现有 `openWorkspaceFile`（`workspace:openFile`）的 `shell` 打开逻辑，按 projectId 解析 `workspace_path` 后打开，避免渲染进程传递任意绝对路径。
- 项目卡片与工作区侧边栏两处路径展示统一改为可点击链接，点击链接时不触发卡片/导航的默认行为（stopPropagation），路径为空时降级为“未设置工作区”且不可点击。
- 不影响现有 scope 文件打开（`workspace:openFile`）与项目增删改流程。
