# 反馈 #22 开发计划：对话后自动更新对话标题

## 背景

反馈 #22 要求“对话后，对话标题要自动更新”。结合当前项目代码，聊天能力由主进程 `src/chat/chatController.js` 负责消息持久化、LLM 调用和工具循环；`src/main.js` 通过 `chat:*` IPC 将事件转发给渲染层；`src/renderer/hooks/useChat.ts` 管理当前项目的对话列表、活跃对话和消息流；侧边栏 `src/renderer/components/workspace/WorkspaceSidebar.tsx` 使用 `chatState.conversations` 渲染对话标题。

当前代码已经有后端自动标题雏形：`chatController` 中的 `generateConversationTitle()` 会在成功完成回复后，基于首条用户消息生成短标题并更新 `conversations.title`。但链路存在断点：`src/main.js` 的 `onDone` 回调只转发 `status/error`，会丢弃 controller 发出的 `conversationId/title`；`ChatDoneEvent` 类型也没有声明标题字段；`useChat` 的完成事件处理只固化流式消息和刷新历史，不会同步会话列表标题。因此数据库标题即使更新，前端侧边栏也不会在对话结束后自动显示新标题。

## 设计约束

- 自动标题只在对话成功完成后生效；错误、中止、达到最大轮次等状态不应伪造标题。
- 不覆盖用户已手动命名的标题；仅允许空标题、`新对话`、`默认对话` 等占位标题被自动替换。
- 标题生成失败不能影响主聊天流程；失败时保留原标题，并正常发送完成事件。
- 前端应在无需刷新页面、无需切换项目、无需手动重进对话的情况下更新侧边栏标题。
- 事件契约保持向后兼容：旧路径只有 `status/error` 时，渲染层仍按当前行为处理完成状态。
- 不改变聊天消息隔离、AI 配置解析、工具调用、手动重命名、置顶排序和删除对话流程。
