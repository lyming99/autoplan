# 修复 Codex CLI 对话模式因 API Key 门禁无法发送消息

## 需求概要

反馈 #3：在 AI 对话中已配置 Codex CLI，并在对话 Composer 选择 Codex 模式后无法进行对话。结合当前代码，后端已存在 codex provider、codexChatBackend 和 chatController 的 Codex 分流，且 Codex 路径不需要 HTTP API Key；主要风险在前端 ChatView/useChat 仍以 hasApiKey 作为输入框、发送按钮、空状态和新建会话默认配置的统一可用性判断，导致 Codex 配置因为没有 API Key 被误判为不可用。需要把“HTTP provider 需要 API Key”和“Codex CLI 本地可用”拆开，确保选择 Codex 后可以发送消息并进入既有 codex exec 后端，同时保持 OpenAI/DeepSeek/Anthropic 缺少 Key 时的阻断行为。
