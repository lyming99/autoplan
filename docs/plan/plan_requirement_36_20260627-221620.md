## 总体验收标准

- 设计一致性：任务页与设置页的布局、信息层级、状态表达、颜色、圆角、阴影、图标和间距与 `ui/tasks.html`、`ui/settings.html` 的目标风格一致，并复用 `ui/workspace.html` 的共享视觉规范。
- 功能不回归：Plan 阅读、并发执行、任务执行/停止、状态筛选、设置保存、CLI 后端配置、scope 文件配置和 MCP 信息展示保持原有业务行为。
- 响应式可用：桌面宽度下优先呈现设计稿双栏结构；窄屏下可读、可操作、无横向溢出，长文本不会撑破卡片或遮挡操作按钮。
- 可访问性可用：关键按钮、分段控件、导航项和表单字段支持键盘访问与可读标签，焦点态清晰，状态不只依赖颜色表达。
- 最终验收命令：`npm run check && npm run build`。
- 验收范围：前端渲染层任务页、设置页、共享样式、相关图标和补充测试代码。
- 通过标准：最终验收命令退出码为 0；手动对照设计稿确认任务页和设置页关键视觉与交互满足需求，且未发现阻断级回归。
