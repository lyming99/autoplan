# AutoPlan v0.2.2

## 近期更新摘要

本版本整合了 beta.6 之后完成的多个计划任务，并完成一轮代码重构。

### 新功能

- **深色主题**：支持浅色/深色/跟随系统三种模式，侧栏底部提供快捷切换，偏好持久化到 localStorage（来源：反馈 #4）
- **脚本模块「定时」筛选 Tab**：脚本列表新增独立「定时」筛选选项，与已有「全部」「触发型」「手动」并列（来源：需求 #17）
- **已完成任务支持重新执行**：已完成/失败/停止的任务可重新点击执行，重新执行时清理 plan 文件旧完成标记；对已完成计划追加任务后会重新激活并继续执行（来源：反馈 #6）

### 修复

- 修复 `useTheme` 文件 JSX 解析失败导致 Vite 开发服务器报错：Provider 文件扩展名与 JSX 内容匹配（来源：反馈 #8）
- MCP 默认 HTTP 端口被占用时自动回退到可用端口；显式指定端口被占用时仍返回可读错误（来源：反馈 #8）
- 修复计划生成 prompt 中智能引号方向错误，导致附件场景未强制要求最终验收节点
- 统一文案「任务与计划」→「计划与任务」（来源：反馈 #7）

### 重构

- 将 `loopService.js` 中的验收、需求/反馈删除、计划生命周期、agent-CLI 运行与配置等逻辑抽离到 `src/loop/` 下独立模块，文件从 2315 行降至 1692 行
- 抽离 `ScriptEditorModal` 的 helper 与脚本编辑弹窗样式到独立文件，缓解相关文件超长

## 完整验收

- `npm run check` 通过（TypeScript 检查 + 语法检查 + 文件长度检查）
- `npm run smoke` 通过
- `npm run build` 通过
