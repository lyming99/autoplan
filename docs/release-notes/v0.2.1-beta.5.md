# AutoPlan v0.2.1-beta.5

> **Beta 未测试版本**：这是面向快速分发的预发布版本，未经过完整人工测试，不等同于稳定正式版，也不承诺零缺陷。

## 近期更新摘要

本次 beta 聚焦把「需求 - 计划 - 执行 - 验收」链路补完整，并把外部接入和本地自动化能力做成可配置、可观察的工作区能力。

- **新增验收模块**：计划和任务新增 `accepted_at` 字段，已完成但未验收的计划/任务会进入独立验收页；支持逐项验收、按计划批量验收、全部验收，以及从最近验收记录中取消验收。
- **新增脚本模块与循环钩子**：工作区可创建 Node/Bash/PowerShell/CMD 脚本，支持内联脚本或文件来源；脚本可手动运行，也可挂到 `plan:after`、`task:after`、`validation:before`、`loop:end`、`on:fail` 等循环阶段，并记录退出码、耗时、日志和事件。
- **MCP 外部接入可视化配置**：设置页新增 MCP 启停、传输方式、监听地址、端口、路径和 Bearer 密钥配置；密钥不再明文回填到界面，支持随机生成、清空和脱敏展示。HTTP 启动失败会给出端口占用、权限、防火墙等更可处置的错误说明。
- **Claude 活动流增强**：Claude CLI 改用 `--output-format stream-json --verbose`，AutoPlan 会解析流式事件生成活动时间线，并从 result 事件提取最终文本和会话 id，降低只读 stdout 文本时丢失执行过程的风险。
- **新增 Oh My Pi CLI 后端**：在 Codex、Claude、OpenCode 之外新增 `oh-my-pi` provider，默认命令为 `omp`，使用 `omp --print` 通过 stdin/stdout 执行单次非交互任务。
- **输入草稿与工作区导航优化**：需求/反馈输入框按项目保存本地草稿；工作区侧栏新增验收和脚本入口，搜索、计划/任务列表、设置页布局与图标体系同步更新。
- **macOS 分发与 MCP 说明补强**：README 补充非沙箱直接分发、首次网络监听弹窗、in-app HTTP 与独立 stdio MCP 接入方式的区别，以及端口受限时的处理方式。
- **设计稿与测试覆盖扩展**：新增验收页、脚本页静态设计稿，并补充 Agent CLI、Claude 活动解析、脚本钩子、MCP 配置/服务、验收状态、工作区页面等单元和冒烟覆盖。

## Beta 风险提示

本 beta 版本用于更快分发近期修复和功能更新。当前策略是：AI 修复问题后，只要项目编译通过即可发布，以减少人工测试和等待时间。因此，下载测试或日常使用中仍可能遇到功能缺陷、平台差异或安装包问题。本次变更涉及数据库迁移、循环钩子执行、MCP 配置重启、多个 Agent CLI 后端和新的验收状态字段，建议先在测试工作区或可恢复的真实工作区中验证。

## 问题反馈

如果你发现问题，请直接到 GitHub Issue 界面提交反馈，并尽量包含：

- 使用的操作系统与安装包类型。
- 选择的 CLI 后端、命令路径和是否已完成 CLI 认证。
- MCP 传输方式、端口、鉴权设置，以及最近错误文案。
- 脚本运行时、触发阶段、退出码、日志或相关事件。
- 复现步骤、预期行为和实际行为。
- 相关截图、日志或附件。

后续 AutoPlan/AI 流程会持续从 Issue 中发现问题，生成修复计划并继续处理。
