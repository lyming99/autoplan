# AutoPlan v0.2.3-beta.22

> **Beta 预发布版本**：本版本用于尽快验证近期功能与稳定性改进，建议重要工作区和数据库先备份后升级。

## 本次更新

- 新增模型 Token 用量统计：统一解析 OpenAI、Anthropic、Codex、Claude 与 Oh My Pi 的用量信息，写入本地数据库，并在工作区概览中展示今日及累计用量。
- 完善计划与任务控制：支持精确停止单个任务、停止整份计划、删除计划，并补充取消归属、状态回收和重复请求保护，降低误停其他运行任务的风险。
- 改进计划展示与文件操作：从 Markdown 提取更清晰的计划标题，支持通过受控的 Go 文件策略打开计划路径。
- 合并最新远程改进：修复任务停止流程，增加项目排序界面，并持久化运行时配置。
- 改进需求与验收流程：同步关联多阶段计划的验收状态，完善计划生成失败筛选和重试相关交互。
- 增强运行时兼容性：补充 Agent CLI 会话元数据和用量解析，扩展 Go HTTP 契约、OpenAPI 与业务冒烟覆盖。
- 修复启动审计误报：数据库时间戳同时存在秒级与毫秒级精度时，审计现在按实际时间顺序比较，避免错误显示 `daemon_readiness_missing`。

## 数据迁移与兼容说明

- 首次启动会执行数据库 v4 迁移并新增模型用量记录结构。
- 升级前建议备份 AutoPlan 数据库。完成 v4 迁移后，不建议直接使用更旧的 sidecar 版本打开同一数据库。
- 继续保持 Go sidecar 单写入者架构；旧配置字段仍按现有兼容规则映射到计划生成与执行配置。

## 已验证

- `npm run build`：TypeScript 与 Vite 生产构建通过。
- `npm run package:win`：Windows x64 NSIS 安装包构建通过。
- Go sidecar 启动、计划/任务停止、计划/项目删除、终端、脚本、验收、Intake 与 MCP 业务冒烟通过。
- Release workflow 使用的数据库迁移、实例锁、sidecar smoke 和产物校验测试通过。
- 本地生成产物：`AutoPlan-0.2.3-beta.22-win-x64-Setup.exe`。

## 已知风险

- 完整的历史 Node 与 Go 测试集合仍包含旧 IPC 传输契约、测试夹具和文件长度规则相关的存量失败；本次发布实际使用的构建、打包、迁移及业务冒烟门禁已通过。
- Windows 安装包已在本地完成端到端打包验证；macOS 与 Linux 产物由版本标签触发的 GitHub Actions 构建，最终状态以 Release workflow 为准。
- 本版本仍是 beta，可能存在尚未覆盖的平台差异或边缘场景。
