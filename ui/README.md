# AutoPlan UI 设计稿

纯 HTML/CSS 设计稿，浏览器直接打开即可预览，无需依赖。

项目和工作区是**两个独立的界面、各自不同的布局**，分属两个文件：

## 文件

| 文件 | 界面 | 布局 |
|------|------|------|
| `projects.html` | **项目管理** | 顶部 header + 全宽内容区（项目卡片网格 + CRUD） |
| `workspace.html` | **工作区** | 左侧边栏导航 + 主区（概览/需求/反馈/任务/事件） |
| `tasks.html` | **任务与计划** | 左侧边栏导航 + 双列（Plan 卡片 / 任务列表） |
| `settings.html` | **设置** | 左侧边栏导航 + 单栏卡片流（循环控制 / scope 打开 / MCP 接入） |

## projects.html · 项目管理
- 顶部 sticky header：品牌 + 「打开工作区」入口 + 「创建项目」
- 页面标题区 + 概览统计（项目总数 / 运行中 / 异常）
- 工具栏：搜索框 + 状态分段筛选（全部/运行中/已停止/异常）
- 项目卡片网格：头像、名称、路径、描述、计划/任务/间隔统计、状态指示灯、更新时间
- 卡片右键菜单：打开工作区 / 编辑 / 复制 / 删除
- 完整 CRUD：
  - **创建/编辑**：模态框（图标色 6 选 1、名称*、路径*、验收命令、CLI 后端、CLI 命令路径、间隔、描述）
  - **复制**：生成 `-copy`，状态置为已停止
  - **删除**：二次确认，提示不删磁盘文件
- 点击卡片 / 「打开工作区」→ 跳转 `workspace.html`

## workspace.html · 工作区
恢复为第一版的工作区布局（侧边栏导航），五个视图间切换：
- **概览** — 统计卡 + 渐变循环状态卡 + 阶段流水线 + 近期事件
- **需求 / 反馈** — 流式列表 + 悬浮圆角输入框 + 附件
- **任务与计划** — 循环控制（含 CLI 后端选择 Codex/Claude）+ 计划草稿 + Plan/Task 网格 + 并发建议入口
- **事件流** — 循环运行日志表格
- **设置** — 工作区 / CLI / scope 打开配置 + MCP 外部接入只读信息
- 侧边栏底部显示当前循环状态与 CLI 后端
- 顶部搜索框支持跨需求、反馈、任务、Plan 和事件检索；未输入关键字时不展示可搜索总数，输入后由命中反馈和结果概览展示统计。

## 需求 / 反馈附件交互
- 需求和反馈输入框支持三种附件添加方式：点击选择文件、拖拽文件到输入区、在输入框内直接粘贴剪贴板图片或截图。
- 粘贴 PNG、JPEG、WebP 等图片后，图片进入待提交附件列表并显示缩略图、文件名和大小；提交前可移除。
- 正文和附件可以一起发送；正文为空时也可以只提交附件，适合快速提交截图类需求或反馈。
- 发送成功后会清空正文和待提交附件；发送失败时保留待提交附件，方便用户重试。
- 附件保存后会以本地文件路径、MIME、大小和 hash 进入计划生成上下文，供 AI 工具按需读取；图片二进制不会直接内联进 prompt。

## CLI 后端选择说明
- 设计稿覆盖项目级 CLI 后端切换：创建/编辑项目与工作区「循环控制」均提供 Codex/Claude 选择。
- 默认仍为 Codex，历史项目未配置 CLI 后端时按 Codex 兼容处理，不改变旧链路交互预期。
- Claude 仅覆盖非交互 `claude -p` 调用链路；使用前需本机已安装 `claude` CLI 并完成认证，必要时在 CLI 命令路径填写完整路径。
- Codex 会话复用仅在 Codex 后端展示和生效；Claude 后端不展示或传递 `codex_session_id`。
- Plan 卡片和 Plan 全文阅读器展示的是计划生成时固化的 CLI 快照，而不是当前项目默认配置；切换默认后端不会让历史计划显示漂移。
- 展示口径统一为 `Codex CLI · 思考深度 medium/high/low` 或 `Claude CLI`；历史计划缺字段时按 Codex + `medium` 降级，Claude 不展示 Codex 思考深度。

## 并发建议与 scope 文件交互
- Plan 卡片展示并发建议摘要：可并发任务数、建议批次数、需串行任务数，以及不可并发原因入口。
- Plan 卡片元信息同时展示计划级 CLI 摘要，与 Plan 全文阅读器摘要保持一致，不从当前项目默认 CLI 反推历史计划。
- 「并发执行」按钮只在存在安全批次且计划未完成、没有运行中任务时可用；点击后先展示确认弹窗，用户确认前不会启动任务，取消不会改变状态。
- 建议批次基于任务 `scope` 冲突检测：scope 互不重叠的 pending 任务可进入同一批；`unknown`、`validation`、空 scope、无法解析 scope 或 scope 重叠任务需串行展示。
- Task 卡片展示从 `scope` 解析出的相关文件列表；长路径折行/省略，缺失文件、`unknown` 和 `validation` 使用不同状态提示，不挤压任务标题与操作按钮。
- Plan 全文阅读器中任务行的 `scope` 文件路径可点击打开；普通 Markdown 链接仍按普通链接渲染，代码块、滚动和表格阅读体验保持不变。
- Plan 全文阅读器摘要区展示计划级 CLI 后端与 Codex 思考深度；读取失败、计划文件缺失或 Markdown 为空时，仍基于计划记录展示该摘要。

## scope 文件打开配置
- 工作区「设置」提供 scope 文件打开方式：系统默认方式、系统文件夹定位、VSCode、第三方编辑器命令。
- 默认使用系统默认方式；选择 VSCode 时命令留空等价于 `code`，也可填写完整路径。
- 第三方编辑器命令支持 `{file}` 占位，例如 `cursor {file}`、`notepad++ {file}`、`zed {file}`。
- 打开前必须校验目标路径位于当前项目工作区内；不存在、目录、越界路径、权限不足或命令失败都需要给出可读提示，不能静默失败。
- 设计上应提醒用户准确填写相对工作区的 scope；无法判断写 `unknown`，完整验收写 `validation`，避免错误并发。

## MCP 外部接入信息
- 工作区「设置」包含只读 MCP 外部接入卡片，不和循环控制表单混在一起，避免误以为 MCP 配置会覆盖项目级 loop 配置。
- 卡片展示服务状态、传输方式、连接地址、访问范围、工具清单和最近错误；默认文案强调 `127.0.0.1` 本机访问，不鼓励公网暴露。
- 工具清单固定展示 `create_project`、`create_requirement`、`create_feedback`，用于用户复制到外部 MCP 客户端配置说明中核对。
- 连接示例优先显示 HTTP 地址 `http://127.0.0.1:43847/mcp`；stdio 模式显示 `npm run mcp:stdio`。
- 错误态显示来自事件流的最近 MCP 启动失败信息；即使 MCP 禁用或失败，GUI 创建项目、提需求和提反馈入口仍保持可用。

## tasks.html · 任务与计划页
双列布局设计稿，从原「任务与计划」分视图拆出来强化 Plan 与任务的关联与并发能力：
- **左列 Plan**：卡片流，顶部状态筛选（全部 / 执行中 / 草稿 / 已完成）。每张 Plan 卡片展示路径、标题、进度条（完成数/总数）、并发建议摘要（可并发数 / 建议批次数 / 串行数，带图标语义色）、计划级 CLI 摘要标签（`Codex · 思考 xhigh`）+ 哈希 + 更新时间，底部操作区（并发执行 / 阅读全文 / 验收状态）。选中态用品牌色描边，运行中/草稿/已完成用左侧色条区分。
- **右列任务**：按 Plan 分组展示（**分组支持折叠/展开**，运行中分组默认展开并高亮、已完成分组默认折叠），顶部状态筛选（全部 / 进行中 / 队列中 / 已完成）。任务卡片展示 task_key + 标题 + 状态芯片 + 耗时（运行中显示「已运行」，已完成显示「耗时」，未开始显示「未开始」）、关联文件链接、scope 文件标签（可打开 / `unknown` / `validation` 用不同语义色区分），底部执行/停止操作。
- **并发执行确认弹层**：按安全批次展示任务清单（task_key + 标题 + scope），批次间标注并发原因，取消不改状态。
- 共享 `workspace.html` 的设计 token 与组件。

## settings.html · 设置页
**左侧分组导航 + 右侧配置内容**的双栏布局（应用级侧边栏之外再嵌一层设置导航），类似 VS Code Settings / Notion 设置。设置导航把原「任务与计划」侧边表单拆成四个可切换分组，每组右侧独立滚动、独立保存条：
- **左侧分组导航**（232px 固定列）：循环控制 / CLI 后端 / scope 文件 / MCP 接入，每项带图标 + 右侧状态徽标（未保存 / Codex / VSCode / 运行中）。
- **循环控制**：分区卡「工作区与节奏」（路径、间隔秒数带「秒」后缀、验收命令）+ 底部 sticky 保存条。
- **CLI 后端**：**分段控件**（Codex / Claude）替代下拉；Codex 思考深度用**三栏选项卡**（Low / Medium / High），切到 Claude 时隐藏思考深度并展示 Claude 说明；CLI 命令路径留空提示随后端变化。
- **scope 文件**：**分段控件**（系统默认 / 文件夹定位 / VSCode / 第三方命令）；仅 VSCode / 第三方命令时展开编辑器命令字段，内联提示强调越界校验与 `unknown`/`validation` 约定。
- **MCP 接入**：只读信息行（服务状态 / 传输方式 / 访问范围 / 连接地址 / 工具清单 / 最近错误），标题旁 MCP 状态徽标，连接地址附可复制条；保留禁用提醒横幅。
- 每个可编辑分组（循环控制 / CLI 后端 / scope 文件）底部都有独立 sticky 保存条；MCP 接入为纯只读无保存条。
- 共享 `workspace.html` 的设计 token 与组件（卡片、芯片、LED、按钮），可直接对照实现。

## 设计说明
- 两个界面共享同一套设计 token（品牌色、语义色、圆角、阴影），保持视觉统一。
- 项目页用**顶栏 + 全宽网格**，工作区用**侧边栏 + 分视图**，布局明显区分。
- 状态语义化：指示灯 LED（脉冲动画）、彩色状态芯片、进度条。
- 配色：在原蓝色基调上引入紫蓝渐变 + 中性灰阶层级，浅色主题。
- 界面图标统一使用本地 inline SVG / icon 片段，不使用 emoji 或字符图形充当图标，确保静态 HTML 离线打开时视觉与 React 界面一致。
