帮我看一下 src/database.js 里的数据库连接是怎么初始化的，有没有用连接池？

已经看了 src/database.js，连接初始化和连接池情况如下：

  • 使用 better-sqlite3 同步驱动，单例模式初始化。
  • 未使用连接池——better-sqlite3 本身是单连接同步执行，原生不支持池化。
  • 通过 DatabaseSingleton 保证全局唯一实例，重复调用 getDatabase() 返回同一句柄。
const db = new Database(dbPath);
db.pragma('journal_mode = WAL');
db.pragma('foreign_keys = ON');

关键点：开启了 WAL 模式提升并发读性能，并启用了外键约束。如果后续需要更高并发，可以考虑迁移到异步驱动。

对话界面 · 设计说明

对应 ChatView.tsx 的全部交互态。输入区采用与需求 / 反馈界面一致的圆角悬浮 composer，模型配置下沉为药丸下拉。所有样式 token 与 workspace.css 保持一致。

1 对话列表（嵌套导航）

折叠在左侧主导航「对话」项下方，缩进展示。默认显示最近 5 个对话，每项含标题 + 模型 + 相对时间，hover 出现重命名 / 删除。当前激活项品牌色高亮 + 左侧圆点。底部「加载更多对话 ↓」展开历史记录。

2 用户气泡

右对齐，蓝底（--brand-600）白字，气泡内部文字保持自然左对齐。支持 Markdown、内联代码与代码块（代码块为半透明白底）。

3 AI 气泡

左对齐，灰底（--bg-soft）配 --text-1 文字，头像显示「AI」。正文支持完整 Markdown 渲染（列表、代码块、链接、表格）。

4 思考折叠区

AI 气泡内嵌的推理过程折叠卡。流式输出时显示「🤔 思考中…」+ 三点跳动动画；完成后变为「✅ 思考完成」，点击展开查看推理细节。

5 工具调用卡片

左侧带品牌色竖条（--brand-400）的卡片。显示工具名 + 参数摘要；⚙ 表示执行中、✓ 表示完成。点击展开查看完整返回结果（JSON）。

6 流式光标

AI 流式输出时，气泡右侧显示闪烁的 ▊ 块状光标，chat-blink 动画 0.8s 循环，提示正在生成。

7 系统消息

居中显示的灰色细字（--text-4），无气泡边框。用于「对话已切换」「已中止生成」等系统级提示。

8 输入区 · Composer

圆角悬浮（--r-xl + --shadow-lg），与需求界面 .composer 同款。无边框 textarea，聚焦时整卡变品牌色边框 + 辉光阴影。底部工具行分左右两组。

9 模型配置下拉

Composer 左侧药丸按钮组（.chat-model-select），三个下拉：供应商（品牌色强调，智谱/OpenAI/Claude…）、模型模式（GLM-4.6 / Air…）、思考深度（关闭/低/中/高）。点击展开原生 select，选中即生效。

10 发送 / 停止按钮

Composer 右侧圆形品牌渐变发送按钮（与需求界面 .send-button 一致）。生成中时变为红色停止按钮（.stop-button），点击中止流式输出。

11 空状态

无消息时居中展示引导。已配置 Key 时提示「开始对话」并说明快捷键；未配置时显示钥匙图标引导前往设置。
